## 2. 文件职责与接口契约

### 2.1 职责

单帧点迹生成函数。对单部雷达生成目标点迹（含检测概率筛选、偏差+噪声）和虚警杂波（泊松分布）。[S][P]

### 2.2 输入输出

| 输入 | 类型 | 含义 |
|---|---|---|
| `rx_lon, rx_lat` | 度 | 接收站位置 |
| `tx_lon, tx_lat` | 度 | 发射站位置 |
| `tgt_lon, tgt_lat` | 度 | 目标位置 |
| `tgt_lon_rate, tgt_lat_rate` | deg/s | 目标速度 |
| `frameID` | int | 帧编号 |
| `time_sec` | s | 仿真时间 |
| `range_bias, az_bias` | m, deg | 系统偏差 |
| `beam_center` | deg | 波束中心 |
| `params` | 结构体 | 全局参数 |
| `range_noise, az_noise` | m, deg | 噪声标准差 |

| 输出 | 类型 | 含义 |
|---|---|---|
| `detList` | struct 数组 | 本帧所有点迹（目标+杂波） |
| `has_target_det` | bool | 是否检测到目标 |

## 3. 逐语句块审查

### 3.1 默认参数（第 102—104 行）

**语句职责。** 若未传入噪声标准差，使用 R1 的值。[S]

**问题记录。** `V02-P2-01`：默认使用 R1 噪声水平。若 R2 调用此函数但未显式传入 `range_noise` 和 `az_noise`，会使用 R1 的参数，导致 R2 噪声估计偏低。当前入口脚本在调用 R2 时显式传入了 R2 噪声，所以实际不触发此问题，但接口设计有隐患。[S]

### 3.2 威力覆盖检查（第 117 行）

**语句职责。** 调用 `radar_coverage_check` 判断目标是否在雷达威力范围内。[S]

**数学正确性。** 覆盖检查基于距离（1000-2000 km）和方位（波束中心 ± 半宽）。[M]

### 3.3 检测概率抽签（第 119—158 行）

**语句职责。** 在覆盖区内以 `P_d` 概率检测目标。[S]

**数学正确性。** `rand() <= params.detection_probability` 是标准的伯努利抽样。[M]

**问题记录。** `V02-P1-01`：`detection_probability=1.0`（当前配置）意味着每帧必检测。注释第 122 行称"P_d = 0.6 意味着平均每 3 帧有 1 帧漏检"，与当前实际参数不符。注释是过时的文档残留。[S]

**问题记录。** `V02-P2-02`：`detList = [detList, det]` 动态数组增长。每帧最多 1 个目标点迹 + `n_false` 个杂波（期望 1.5），影响可忽略。但在 MC 循环中（200-500 次 × 52 帧），若 `P_d<1` 导致大量空帧，动态增长的累积开销不可忽视。[S]

### 3.4 天波量测计算（第 127—135 行）

**语句职责。** 调用 `skywave_geometry` 计算群距离、方位和多普勒真值。[S]

**数学正确性。** 与 UKF 端共享同一几何模型，保证仿真-估计一致性。[M]

**性能。** 3 次 `skywave_geometry` 调用（group_range、azimuth、doppler）。每次内部约 5-10 次三角函数。[S]

### 3.5 偏差和噪声注入（第 140—142 行）

**语句职责。** `量测 = 真值 + 固定偏差 + 高斯噪声`。[S]

**数学正确性。** 标准加性误差模型。多普勒不注入系统偏差（注释称"多普勒测量不受距离/方位标定误差影响"），这是合理的物理假设。[M]

### 3.6 目标点迹结构体组装（第 151—156 行）

**语句职责。** 包含原始量测、校正后量测、真值、经纬度、is_clutter 等字段。[S]

**代码质量。** 字段命名清晰，但结构体字段过多（16 个），部分字段在后续处理中未使用（如 `range_true` 仅在评估中使用）。[S]

### 3.7 杂波泊松生成（第 177 行）

**语句职责。** `n_false ~ Poisson(lambda)`，`lambda = 1500 × 0.001 = 1.5`。[S]

**数学正确性。** 泊松点过程是杂波的标准统计模型。[M]

### 3.8 杂波极坐标采样（第 182—201 行）

**语句职责。** 在接收站极坐标中均匀采样杂波位置，再正算到经纬度。[S]

**数学正确性。** 极坐标采样确保杂波严格在威力覆盖区内。[M]

**问题记录。** `V02-P2-03`：杂波多普勒 `fake_vr = -200 + rand()*400` 在 [-200, +200] 均匀分布。但真目标的多普勒是连续变化的（由运动学决定），帧间变化通常 < 5 m/s。杂波 Vr 与真目标 Vr 的分布差异是 Vr 门过滤杂波的依据。若实际杂波 Vr 分布更窄（如电离层杂波通常集中在 ±50 m/s），当前 400 m/s 范围高估了杂波 Vr 分散性，使 Vr 门过滤效果偏乐观。[M]

### 3.9 杂波偏差注入（第 220—221 行）

**语句职责。** 杂波也掺入系统偏差，保证校正后一致性。[S]

**数学正确性。** 杂波 `prange = fake_Rg + range_bias`，校正后 `drange = prange - dr_est ≈ fake_Rg`。[M]

**问题记录。** `V02-P2-04`：杂波 `paz = fake_az + az_bias`，但 `fake_az` 本身已是接收站到杂波点的球面方位角。加上 `az_bias` 后，校正减去 `da_est` 时，假设 `da_est` 收敛到 `az_bias`。若偏差估计不准确，杂波校正后仍有残余偏差。[S]

### 3.10 杂波结构体组装（第 219—227 行）

**语句职责。** 杂波点迹字段与目标类似，但 `range_meas` 等为 NaN。[S]

**代码质量。** `radial_vel_meas` 对杂波赋值为 `fake_vr`（非 NaN），与注释"NaN（杂波）"不一致。[S]

## 5. 文件级结论

- **数学可信度**：天波量测模型与 UKF 端共享，仿真-估计一致性良好。泊松杂波模型标准。杂波 Vr 分布 [-200, +200] 可能高估实际电离层杂波的多普勒展宽。
- **代码质量**：过时注释（P_d=0.6）残留；杂波 `radial_vel_meas` 字段赋值与注释不一致；默认噪声 R1 的接口隐患。
- **性能**：3 次 `skywave_geometry` 调用/帧；动态数组增长在 MC 中累积。
- **测试充分性**：端到端冒烟通过，但杂波 Vr 分布假设对 Vr 门性能的影响未获独立验证。
- **剩余未验证项**：不同 `P_d` 值下杂波 Vr 分布对关联率的影响。
