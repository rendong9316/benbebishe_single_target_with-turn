## 2. 文件职责与接口契约

### 2.1 职责

PDA（概率数据关联）权重计算函数。根据波门内点迹集计算各候选量测的关联概率 `beta_vec`，并用这些概率对 3D 新息进行加权，输出加权新息 `innov_weighted`。不含任何 UKF 更新数学（`K`、`P` 更新由 `ukf_jichu('update', ...)` 完成）。[S][P]

### 2.2 输入输出与单位契约

| 输入 | 类型 | 单位 |
|---|---|---|
| `dets_in_gate` | cell 数组，波门内点迹 | — |
| `z_pred` | `[range; azimuth; radial_vel]` | m, deg, m/s |
| `P_zz` | 3×3 量测协方差 | m², deg², (m/s)² 混合 |
| `params` | 参数结构体 | — |

| 输出 | 类型 | 含义 |
|---|---|---|
| `innov_weighted` | 3×1 | PDA 加权 3D 新息 |
| `beta_vec` | 1×m | 各点迹关联概率 |
| `nis_2d` | 标量 | 最高权重量测的 2D 马氏距离 |

### 2.3 统计契约

- 关联概率 `beta_vec` 基于 2D 新息（距离+方位）计算，但用于 3D 新息（含径向速度）加权。
- 返回的 NIS 是 2D 马氏距离，不是 3D 更新 NIS。
- `Pg`（门内概率）和 `Pd`（检测概率）用于归一化常数 `b` 的计算，但当前 `Pg=0.8647` 对应阈值 4，而实际关联门可能是 72。[S][M]

## 3. 逐语句块审查

### 3.1 函数签名与防御性 P_zz 检查（第 22—43 行）

**语句职责。** 检查 `P_zz` 是否为 3×3；若不是则回退到简单新息计算。[S]

**数学正确性。** 当 `P_zz` 维度异常时，函数放弃 PDA 统计，退化为取第一个点迹的新息（或零向量）。这相当于将 PDA 退化为 NN（甚至退化到无新息）。[M]

**代码质量。** `isequal(size(P_zz), [3,3])` 是精确尺寸检查。如果 `P_zz` 是 2×2（仅距离+方位），函数会直接回退，而不是使用 2D 子矩阵。这可能是一个 bug：`nn_associate` 返回的 `P_zz_2d` 是 2×2，但 `pda_weight` 的签名要求 3×3。[S]

**问题记录。** `V04-P1-01`：`P_zz` 维度检查过于严格（要求 3×3），当传入 2×2 矩阵时直接回退。应支持自适应维度或使用 3×3 完整矩阵。[S]

### 3.2 2D 子矩阵提取（第 44 行）

**语句职责。** 提取 `P_zz(1:2, 1:2)` 用于 2D 马氏距离计算。[S]

**数学正确性。** `P_zz_2d` 是距离和方位的协方差子块。[M]

### 3.3 3D 量测提取与 2D 新息计算（第 46—60 行）

**语句职责。** 对波门内每个点迹提取 3D 量测 `[drange; daz; radial_vel_meas]`，计算 2D 新息和方位角包裹。[S]

**数学正确性。**

```matlab
innov_2d(:, i) = z_meas_3d(1:2, i) - z_pred(1:2);
% 方位角包裹
if innov_2d(2, i) > 180, innov_2d(2, i) -= 360; end
mahal_2d(i) = innov_2d(:, i)' * (P_zz_2d \ innov_2d(:, i));
```

标准 2D 平方马氏距离。[M]

**代码质量。** 方位角包裹正确。[S]

### 3.4 PDA 归一化常数计算（第 62—70 行）

**语句职责。** 计算归一化常数 `b`。[S]

**数学正确性。**

```text
alpha = Pd * Pg
V_norm = 2*pi*sqrt(det(P_zz_2d))
b = lambda * V_norm * (1 - alpha) / alpha
```

标准 PDA 归一化常数公式。[M]

**问题记录。** `V04-P1-02`：`V_norm` 使用 2D 行列式，但 `P_zz` 是 3×3。标准 PDA 中 `V_norm` 应该是量测空间体积。如果关联在 2D 空间（距离+方位）进行，2D 行列式是正确的；但如果后续 3D 新息加权涉及径向速度，体积应在 3D 空间计算。当前实现混用 2D 关联概率和 3D 新息，归一化常数的维度语义不清。[M]

### 3.5 关联概率计算（第 72—80 行）

**语句职责。** 计算指数似然并归一化。[S]

**数学正确性。**

```text
e(i) = exp(-0.5 * mahal_2d(i))
beta_vec(i) = e(i) / (b + sum(e))
```

标准 PDA 权重公式。[M]

**问题记录。** `V04-P2-01`：`e(i)` 是马氏距离的高斯核，但标准 PDA 中 `e(i)` 应该是 2D 卡方 PDF 在 `mahal_2d(i)` 处的值，即 `1/(2*pi*sqrt(det(P_zz_2d))) * exp(-0.5*mahal_2d(i))`。当前实现省略了归一化系数 `1/(2*pi*sqrt(det))`，这等价于假设 `det(P_zz_2d) = 1/(2*pi)`。当 `P_zz_2d` 的行列式偏离 1 时，`beta_vec` 的相对权重不变，但绝对质量（与 `b` 的比较）会失真。[M]

### 3.6 3D 加权新息构造（第 82—92 行）

**语句职责。** 用 2D 计算出的 `beta_vec` 对 3D 新息加权。[S]

**数学正确性。**

```matlab
innov_3d(:, i) = z_meas_3d(:, i) - z_pred;
innov_weighted = innov_3d * beta_vec';
```

**关键问题**：`beta_vec` 基于 2D 距离+方位的马氏距离计算，但应用于 3D 新息（含径向速度）。当两个候选量测距离和方位接近但径向速度方向相反时，它们的 `beta_vec` 可能几乎相同，导致加权新息的方向被径向速度错误地拉偏。[M]

**问题记录。** `V04-P1-03`：2D 关联概率应用于 3D 新息加权，维度不匹配。当候选量测在 2D 空间接近但 3D 空间分离时（如方位相近但径向速度相反），PDA 权重无法区分，导致速度方向被错误加权。[S][M]

### 3.7 2D NIS 返回（第 94—96 行）

**语句职责。** 返回最高权重量测的 2D 马氏距离。[S]

**数学正确性。** `nis_2d = mahal_2d(best_idx)` 是 2D 平方马氏距离，自由度 `k=2`。[M]

**问题记录。** `V04-P2-02`：返回 2D NIS 供上层诊断使用，但上层可能将其解释为 3D NIS（自由度 `k=3`）。若用 `chi2cdf(nis_2d, 3)` 解释，覆盖率会被严重低估。[S]

## 5. 文件级结论

- **数学可信度**：PDA 权重计算公式形式正确，但 2D 关联概率应用于 3D 新息加权存在维度不匹配。归一化常数 `b` 使用 2D 体积但应用于 3D 新息，语义不清。
- **代码质量**：`P_zz` 维度检查过于严格，2×2 输入直接回退。`beta_vec` 省略 PDF 归一化系数不影响相对权重但影响绝对质量。
- **性能**：三重循环（m 个点迹 × 每个点迹 2D 计算 + 3D 计算），`m` 通常很小（0-5），性能可忽略。
- **测试充分性**：端到端冒烟通过，但 2D/3D 维度不匹配的具体影响未获独立验证。
- **剩余未验证项**：不同 `Pg` 值对 `beta_vec` 绝对质量的影响；2D/3D 维度不匹配在径向速度冲突场景的数值表现。
