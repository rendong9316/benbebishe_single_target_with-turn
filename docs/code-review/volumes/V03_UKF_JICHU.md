## 2. 文件职责与接口契约

### 2.1 职责

该文件是 UKF 基础滤波器模板，提供创建、初始化、预测准备、更新、纯预测、量测模型、Sigma 点生成、状态转移（CV/CT）、极坐标反解和协方差正则化。它不含任何关联或航迹管理逻辑，是 `single_track_runner` 和 `ukf_dispatch` 的数学内核。[S][P]

### 2.2 输入输出与单位契约

| action | 输入 | 输出 | 单位 |
|---|---|---|---|
| `create` | 参数、雷达/Tx 经纬度、采样间隔 | UKF 模板结构体 | 经纬 deg, dt s, R m, Q 经验 |
| `init` | 当前模板、1 或 2 个量测 | 填充 `ukf.x`, `ukf.P` | 同上 |
| `prepare` | 当前模板 | `x_pred, P_pred, X_pred, z_pred, Z_pred, P_zz, ukf` | 状态 deg/deg/s, 量测 m/deg/(m/s) |
| `update` | 模板, PDA 加权新息 | `lon, lat, ukf` | deg |
| `predict` | 模板 | `x_pred, P_pred, X_pred, ukf` | 同上 |
| `measurement` | 模板, 状态 | 量测向量 `[Rg; az; vd]` | m, deg, m/s |

状态顺序 `[lon; lon_rate; lat; lat_rate]` 混合角位置和角速度。量测 `Rg` 为 m，`az` 为 deg，`vd` 为 m/s。协方差矩阵 `P` 和 `Q` 的元素单位为对应状态分量的乘积，`R` 为对应量测分量的乘积。[S][M]

### 2.3 模型边界

- 默认 CV 模型；IMM 场景可覆盖为 CT。
- 天波传播模型假设固定球体 `R_e=6371000 m` 和固定电离层虚高 `H=300000 m`。
- 反解迭代使用 `skywave_geometry`，与仿真端共享同一几何模型。[S][P]

### 3.2 `create_ukf` — 滤波器模板创建（第 46—96 行）

#### 3.2.1 基础参数保存（第 47—53 行）

#### 3.2.2 UKF 核心参数计算（第 55—70 行）

**语句职责。** 计算 `n=4`, `alpha`, `beta`, `kappa`, `lam` 及 Sigma 点权重。[S]

**数学正确性。**

```text
lam = alpha^2 * (n + kappa) - n
```

标准 UKF 公式。`Wm` 和 `Wc` 初始化为均匀分布 `1/(2(n+lam))`，中心点额外加 `lam/(n+lam)`（均值）和 `(1-alpha^2+beta)`（协方差）。[M]

当前参数：`alpha=params.ukf_alpha`（通常为 1e-2 或 0.5），`beta=2.0`（高斯最优），`kappa=params.ukf_kappa`（通常为 0）。当 `alpha=1e-2` 时，`lam ≈ -3.9996`，`n+lam ≈ 0.0004`。

**关键数值问题：**

```text
Wm(1) = lam/(n+lam) ≈ -9999
Wm(2:9) = 1/(2*0.0004) ≈ 1250
Wc(1) = -9996
Wc(2:9) = 1250
```

中心点权重约 -9999，其余各约 1250。负权重大量放大中心点的数值误差，且 `Wc(1)` 与 `Wm(1)` 差值 `1-alpha^2+beta ≈ 2` 相对权重尺度极小。这导致：

1. Sigma 点传播后，中心点在均值和协方差计算中占主导，近似退化为 EKF 的一阶展开行为。
2. 当 `alpha` 极小时，`(n+lam)` 接近零，`1/(2(n+lam))` 可能溢出或产生极大数值。[M]

**代码质量。** 参数 `alpha` 通过 `params` 传入，但无范围检查。`alpha=0` 时 `lam = kappa - n`，若 `kappa=n` 则 `lam=0`，`Wm` 分母为零。[S]

**问题记录。** `V03-P1-01`：`alpha=1e-2` 时负权重约 -9999，UKF 退化为近似 EKF，且数值稳定性差。[M]
`V03-P2-01`：`alpha` 无范围检查，`alpha=0` 时分母为零。[S]

#### 3.2.3 量测噪声协方差 R（第 72—75 行）

**语句职责。** 构建 3×3 对角 `R`。[S]

**数学正确性。** 假设三类量测噪声互不相关，符合 `skywave_geometry` 中独立实现。[S][M]

**代码质量。** 从 `params` 读取，字段名带单位后缀 `_std_m`、`_std_deg`、`_std_ms`，清晰。[S]

#### 3.2.4 过程噪声协方差 Q（第 77—79 行）

**语句职责。** 经验对角矩阵乘场景缩放因子。[S]

**数学正确性。**

```text
Q_base = diag([1e-9, 1e-13, 1e-9, 1e-13])
Q = Q_base * ukf_Q_scale
```

速度方向噪声比位置方向小 1000 倍。对 CV 模型，`Q` 应代表状态变化的方差增量；`1e-9 deg²/s ≈ 0.1 m²/s²`（赤道处），`1e-13 deg²/s³ ≈ 0.001 (m/s²)²`。速度方向噪声过小意味着滤波器极"紧"地约束速度估计，转弯时难以快速响应。[M]

**问题记录。** `V03-P1-02`：速度方向 `Q` 过小，转弯场景下 CT 模型也难以弥补。建议基于连续 Wiener 过程模型设计。[M]

### 3.3 `init_ukf` — 两点初始化（第 102—152 行）

#### 3.3.1 位置反解（第 103—110 行）

**语句职责。** 调用 `meas_to_latlon_ukf` 从极坐标量测反解经纬度。[S]

**数学正确性。** 两点起始时优先使用 `meas2`（当前帧），避免过时量测。[S]

#### 3.3.2 两点差分速度（第 112—141 行）

**语句职责。** 若两量测帧间隔 ≤ 2，计算差分速度；否则回退到 `v=0`。[S]

**数学正确性。**

```text
dlat_m = (lat2-lat1) * 111320
dlon_m = (lon2-lon1) * 111320 * cos((lat1+lat2)/2)
speed = sqrt(dlat_m^2 + dlon_m^2) / dt
```

固定换算系数 `111320 m/deg` 在赤道处精确，但在纬度 33° 处经度实际距离约为 `111320 * cos(33°) ≈ 93300 m/deg`，即此处经度距离被高估约 19%。[M]

速度检查 `[50, 500] m/s` 覆盖亚音速到超音速，合理。[M]

**问题记录。** `V03-P2-02`：固定 `111320` 换算系数未考虑纬度相关修正（`cos(lat)`），经度方向速度高估约 20%。[M]

#### 3.3.3 协方差重置（第 145—148 行）

**代码质量。** 两处 `P` 初始化代码重复，应统一为单一辅助函数。[S]

### 3.4 `prepare_ukf` — 预测+量测统计（第 159—182 行）

#### 3.4.1 预测步骤（第 161 行）

**语句职责。** 调用 `predict_step_ukf` 获取先验。[S]

#### 3.4.2 量测预测（第 163—177 行）

**语句职责。** 对预测状态和所有 Sigma 点调用 `measurement_ukf`，计算量测均值 `z_pred` 和协方差 `P_zz`。[S]

**数学正确性。**

```text
z_pred = h(x_pred)
P_zz = R + Σ_i Wc_i * (h(X_i) - z_pred)(h(X_i) - z_pred)'
```

标准 UKF 量测统计。[M]

**NaN 守卫。** 若任何 `Z_pred` 为 NaN（可能来自 `meas_to_latlon_ukf` 迭代发散或 `skywave_geometry` 输入异常），回退到 `P_zz = R`。合理。[S][M]

**性能。** 9 次 `skywave_geometry` 调用（1 次 `z_pred` + 8 次 `Z_pred`）。每次调用内部包含 `geocentric_angle`、`slant_range`、`doppler` 等计算。在 MC 循环中此开销显著。[S]

#### 3.4.3 缓存（第 180—181 行）

**代码质量。** 结构体缓存合理。[S]

### 3.5 `update_with_innov` — Kalman 更新（第 191—246 行）

#### 3.5.1 纯预测帧（第 201—208 行）

**语句职责。** `innov_w=[]` 时保留预测状态，不更新。[S]

**数学正确性。** 纯预测帧跳过更新是合理的关联回退路径。[M]

#### 3.5.2 互协方差 P_xz（第 210—216 行）

**语句职责。** 标准 UKF 互协方差计算。[S]

**数学正确性。** `P_xz = Σ Wc_i * (X_i-x_pred)(Z_i-z_pred)'`。[M]

#### 3.5.3 卡尔曼增益 K（第 218—223 行）

**语句职责。** `K = P_xz / P_zz`，失败时用伪逆。[S]

**数学正确性。** MATLAB 右除 `/` 等价于 `P_xz * inv(P_zz)`，数值稳定性优于显式求逆。[M]

#### 3.5.4 状态更新（第 225—226 行）

**语句职责。** `x_new = x_pred + K * innov_w`。[S]

**数学正确性。** 标准 Kalman 更新。[M]

#### 3.5.5 NaN/Inf 守卫（第 228—235 行）

**语句职责。** 更新后检查数值异常，回退到预测。[S]

#### 3.5.6 协方差更新（第 237—241 行）

**语句职责。** `P_new = P_pred - K * P_zz * K'`，然后正则化。[S]

**数学正确性。** 标准 Joseph 形式简化版 `P = P_pred - K*S*K'`。但标准 Joseph 形式应为：

```text
P = (I - K*H) * P_pred * (I - K*H)' + K*R*K'
```

当前实现缺少 `(I-KH)` 结构的对称性保证，在 `P_zz` 非正定时可能导致 `P_new` 不定。正则化 `regularize_cov_ukf` 事后修补，但正则化后的 `P` 不再等于任何理论后验协方差。[M]

**问题记录。** `V03-P1-03`：协方差更新使用 `P_pred - K*P_zz*K'` 而非 Joseph 形式，数值稳定性依赖于事后正则化。[M]

### 3.7 `measurement_ukf` — 量测模型（第 295—315 行）

### 3.8 `sigma_points_ukf` — Sigma 点生成（第 321—340 行）

**语句职责。** Cholesky 分解生成 `(n+lam)*P` 的平方根，构造 2n+1 个 Sigma 点。[S]

**数学正确性。** 标准 UKF Sigma 点构造。[M]

**数值稳定性。** Cholesky 失败时加 `1e-8*I` 扰动，是标准数值技巧。[M]

**问题记录。** `V03-P2-04`：Cholesky 失败时扰动量 `1e-8` 相对 `P` 的量级可能过大或过小；建议使用相对扰动如 `1e-8 * max(diag(P))`。[M]

### 3.9 `state_transition_ukf` — CV 模型（第 346—352 行）

**语句职责。** 4×4 状态转移矩阵。[S]

**数学正确性。** 经纬度状态下的 CV 模型，`lon_next = lon + dt * lon_rate`，`lat_next = lat + dt * lat_rate`。注意这是角坐标线性推进，不是米制笛卡尔 CV。[M]

### 3.10 `state_transition_ct_ukf` — CT 模型（第 369—381 行）

**语句职责。** 协调转弯模型，已知转弯率 `omega`。[S]

**数学正确性。** 转移矩阵形式：

```
F = [1, sin(wT)/w, 0, -(1-cos(wT))/w;
     0, cos(wT),    0, -sin(wT);
     0, (1-cos(wT))/w, 1, sin(wT)/w;
     0, sin(wT),    0, cos(wT)]
```

当 `omega→0` 时，`sin(wT)/w → T`，`(1-cos(wT))/w → 0`，矩阵退化为 CV 形式。[M]

**问题记录。** `V03-P2-05`：`omega=0` 时无除零保护；当前通过 `abs(omega)>1e-12` 条件分支规避，但若 `omega` 极小（如 1e-13），CT 分支仍会执行且数值不稳定。[S]

### 3.11 `meas_to_latlon_ukf` — 极坐标反解（第 388—433 行）

**语句职责。** 经典双基地反解初值 + 天波模型迭代精化 + 球面正算。[S]

**数学正确性。**

```text
baseline = 2*R_e * atan2(sqrt(a), sqrt(1-a))  % Haversine 弦长
phi = az - tx_az                               % 方位角差
r1 = 0.5 * (rng^2 - baseline^2) / (rng - baseline*cos(phi))  % 双基地椭圆反解
```

经典双基地反解公式，假设 Tx-Rx 基线为直线，适用于地表近似。天波斜距 `rng` 包含电离层路径，但反解使用地表基线，这会引入系统偏差。[M]

迭代精化：

```text
tgt = destination_point(radar, r1, az)
Rg_pred = skywave_geometry(tx, radar, tgt)
err = rng - Rg_pred
r1 = r1 * rng / Rg_pred
```

定点迭代 30 次，收敛阈值 1 m。[M]

**问题记录。** `V03-P2-06`：定点迭代 30 次收敛到 1 米精度，对实时跟踪而言过度设计。建议减少到 10 次或放宽到 100 米阈值（对应 0.001° 精度）。[S][M]
`V03-P1-04`：双基地反解初值使用地表基线，但 `rng` 是天波斜距（含电离层路径），初值存在系统性偏差，迭代可能不收敛或收敛到错误解。[M]

### 3.12 `regularize_cov_ukf` — 协方差正则化（第 339—466 行）

**语句职责。** 对称化 + 特征值裁剪 + NaN/Inf 守卫。[S]

**数学正确性。**

```text
P_sym = (P + P') / 2
d_clip = max(d, min_allowed)  % min_allowed = max(min_eig, 1e-6 * max(d))
P_reg = V * diag(d_clip) * V'
```

双阈值策略合理：既防止绝对过小（`min_eig=1e-12`），又保持相对尺度（`1e-6 * max_eigenvalue`）。[M]

**性能。** `eig` 分解 4×4 矩阵，`O(1)`。[S]

**问题记录。** `V03-P3-02`：NaN/Inf 守卫直接返回单位阵，丢失所有先验信息；建议返回初始 `P` 或 `diag([0.05^2, 0.004^2, 0.05^2, 0.004^2])`。[S]

## 5. 文件级结论

- **数学可信度**：UKF 核心数学（Sigma 点、UT 变换、Kalman 更新）形式正确，但 `alpha=1e-2` 导致的极端负权重使滤波器实质退化为近似 EKF，且数值稳定性差。协方差更新缺少 Joseph 形式的对称性保证。反解初值使用地表基线处理天波斜距存在系统性偏差。
- **工程质量**：dispatcher 错误消息格式不完整；`P` 初始化代码重复两处；`alpha` 无范围检查；CT 模型 `omega=0` 边界保护依赖外部条件。
- **性能风险**：`skywave_geometry` 调用未向量化，每帧最多 12 次调用（9 次在 `prepare` + 3 次在 `measurement`），在 MC 循环中是主要热点。
- **测试充分性**：端到端冒烟通过，但 UKF 局部公式（UT 权重和、Sigma 点传播、反解收敛）未获独立函数级运行验证。
- **剩余未验证项**：CT 模型在转弯场景下的数值表现；`alpha` 不同取值对滤波稳定性的影响；反解迭代在不同 `rng` 和 `az` 下的收敛性。
