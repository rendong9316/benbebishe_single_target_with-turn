  三、部署方向：oracle 前序 + 真实三部曲

  阶段 A：起始（truth-seeded initiation）

  - 每个真值目标对应一个 truth_idx，在 t=0 帧用真值状态 + 一次实际测量噪声抽样初始化滤波器
  - 航迹生命周期内始终携带 truth_idx 标签，作为 oracle 关联的钥匙
  - 关闭虚警自由起始路径：所有 confirmed 航迹必须来自真值种子。虚警点允许进入 tentative 池但不允许独立成 confirmed
  航迹（避免"航迹数 > 目标数"污染评估）
  - 单目标自然是 truth_idx=1 一条航迹；多目标是 {1,2,3}，完全统一

  阶段 B：逐帧关联（truth-index association，核心）

  每帧的处理逻辑（对每条活航迹 T，绑定 ac = T.truth_idx）：

  候选 = 本帧所有 det 中 truth_idx == ac 的检测
  if 候选非空：
      若仅 1 个：直接关联
      若 >1 个：选距离 T 当前预测位置最近的（极罕见，只在噪声波动时出现）
      T.miss = 0, T.quality++
  else:
      T.miss++                  % 这才是 Pd=0.6 真正起作用的地方
      T 滤波器纯外推（无测量更新）

  关键性质：
  - 交叉时 A 的检测 truth_idx=1，B 的 truth_idx=2，永远不会跨目标错关联——coalescence 从机制上消失
  - 虚警 truth_idx=0，被自然过滤掉，不污染任何航迹
  - 漏检帧（候选为空）滤波器纯外推——Pd=0.6 的统计特性完整保留
  - 不需要 JPDA 的假设展开、不需要 motion gate、不需要 PDA 加权——这些复杂性全是用来对付"不知道哪个检测是真的"，现在
  oracle 直接告诉你了

  阶段 C：终止（Pd=0.6 自然断裂）

  - T.miss >= K_loss（如 K_loss=5）→ 航迹转 HISTORY/终止
  - 连续 5 帧漏检的概率是 0.4^5 ≈ 1%，所以正常跟踪下不会误终止；只有在真值目标长时间低仰角、波束边缘时才会自然断裂
  - 终止是完全真实的：Pd=0.6 下这本来就会发生

  阶段 D：重起始（truth-aided re-acquisition）

  - 航迹终止后，下一帧检测中若再次出现 truth_idx == ac 的检测 → 重新起始一条新航迹，继承 truth_idx
  - 这模拟了"目标重新进入良好检测区，操作员重新确认起始"
  - 评估时这条新航迹会和原航迹拼成"段"，每段 RMSE 单算，最后做段聚合

  阶段 E：虚警的归宿

  - 虚警检测 truth_idx=0，不参与任何航迹关联
  - 是否允许虚警形成短命 tentative
  航迹？建议完全关闭——既然前序不是研究对象，让虚警仅作为"检测列表里的干扰项存在过"即可，不进入航迹层面
  - 这样 n_tracks_R1 == n_targets（除去短暂断裂期），评估干净

  二、空间分集恢复的数学基础

  单雷达断裂概率：
  - 连续 K 帧漏检概率：P_break_single = (1-Pd)^K = 0.4^5 ≈ 1%
  - 3600s / 30s = 120 帧，期望断裂次数约 120 * 0.01 ≈ 1.2 次/小时
  - 单雷达几乎必然断裂

  双雷达同时断裂概率（假设独立）：
  - 连续 K 帧同时漏检：P_break_dual = (1-Pd)^(2K) = 0.4^10 ≈ 0.01%
  - 期望同时断裂次数约 120 * 0.0001 ≈ 0.012 次/小时
  - 双雷达几乎不会同时断

  这就是融合的物理基础：两雷达的断裂点几乎不重合，互补性极强。任何时刻至少一部雷达在跟，融合就能把整段时间补齐。

  ▎ 前提假设：R1、R2 漏检事件独立。天波超视距雷达下这个假设基本成立（电离层衰落、波束覆盖、目标姿态对不同收发对独立），
  ▎ 如果实际有相关性，恢复能力会打折——这可以在实验中用同时断裂率验证。

  - 输出：匹配/不匹配 + 重合质量分数
  - 关键改进：用比例而非绝对帧数，长短段可匹配

  模块 2：段聚合器（tracklet_grouper）——把同一物理目标的段聚成 group
  - 建二部图：R1 段集合 ↔ R2 段集合，边由模块 1 给出
  - 连通分量 = 一个物理目标
  - 一个 group 可能含：R1 的 3 段 + R2 的 2 段
  - 传递性聚合：R1-A 匹配 R2-X，R1-B 也匹配 R2-X → A、X、B 同组（X 是桥梁）
  - 这个 group 在时间轴上覆盖目标的完整生命周期（被两雷达互补填满）

  模块 3：桥接融合器（bridging_fusion）——在 group 层级做时序融合
  - 输入：group 内所有段的时序状态 + 协方差
  - 时间轴分三类区间：
    - A 类：两雷达都有——标准融合（SCC/BC/CI/FCI），最优区间
    - B 类：仅一部雷达有——用该雷达段做单站估计，相当于退化融合
    - C 类：两雷达都断——纯动力学外推（短）或留空（长），C 类应极罕见
  - 输出：完整时间轴上的融合估计 + 每帧来源标志（A/B/C）

  4.2 重起始 ID 的处理（回应你的"不开上帝视角"）

  完全同意你的原则。落地方式：

  - 每条段有独立 ID，不继承、不预知
  - 段聚合器纯靠段间的时空一致性判断"是否同一目标"
  - 不读取 truth_idx 来辅助聚合（保留 truth_idx 仅用于评估端验证聚合正确性）
  - 段的产生完全自然：跟踪器不知道"这是老目标的延续"，它就是看到检测就起始、连续漏检就终止

  这样设计后，"段聚合"本身就是你三部曲里"匹配"的一部分——而且比单雷达的 dualgate 更接近真实工程价值。

  4.3 段聚合中的歧义处理

  目标交叉时会出现：A 目标的某段和 B 目标的某段在交叉瞬间距离很近，可能被误判匹配。

  应对策略（按优先级）：
  1. 整段统计而非瞬时：段匹配判据用整段平均距离/方向，交叉瞬间只占几帧，不会主导整段统计
  2. 方向一致性硬门：两段的平均速度向量夹角 < 阈值（如 60°）才允许匹配
  3. 多假设保留：若 R1 段同时匹配 R2 段 X 和 Y，先都进候选，融合时用最近邻或权重软决策
  4. 后处理互斥：最终每个 group 限制为同一物理目标，若 group 内 R1 段或 R2 段数异常多（如 ≥3），检查是否误聚合

  4.4 单/多目标统一性

  n_targets=1 时：
  - R1 可能在 1 小时内断成 2 段，R2 断成 1 段
  - 聚合器把 R1 的 2 段 + R2 的 1 段聚成一个 group（通过 R2 那一段作为桥梁）
  - 融合输出覆盖完整 1 小时
  - 机制与 multi 完全一致——单目标再次成为多目标的退化

  这又一次印证你最初的"统一入口"理念：段聚合 + 桥接融合是统一架构的核心机制，不是 multi 特有。

  五、评估口径的调整

  当前评估是"航迹 vs 真值"，新架构下要改成"group vs 真值"：

  ┌────────────────┬─────────────────────────────────────────────────────────────────────┐
  │      指标      │                                定义                                 │
  ├────────────────┼─────────────────────────────────────────────────────────────────────┤
  │ Group 覆盖率   │ group 融合输出帧数 / 真值生命周期帧数                               │
  ├────────────────┼─────────────────────────────────────────────────────────────────────┤
  │ Group RMSE     │ group 融合输出帧 vs 真值的 RMSE（按帧类型 A/B/C 分别统计）          │
  ├────────────────┼─────────────────────────────────────────────────────────────────────┤
  │ 段聚合正确率   │ group 内段集合 vs 该真值对应的所有真值段（开发期用 truth_idx 验证） │
  ├────────────────┼─────────────────────────────────────────────────────────────────────┤
  │ A/B/C 区间比例 │ 融合质量分布，C 类比例越低越好                                      │
  └────────────────┴─────────────────────────────────────────────────────────────────────┘

  这样评估既反映"三部曲极限性能"（A 类 RMSE），也反映"空间分集恢复能力"（B 类填充量），还反映"系统局限"（C 类空洞）。

  一、领域定位：这个问题叫什么名字？

  文献中没有统一术语，同一问题在不同领域有不同称呼：

  ┌────────────────────────────────────────┬─────────────────────────────────┬────────────────┐
  │                  术语                  │            典型领域             │ 对应我们的场景 │
  ├────────────────────────────────────────┼─────────────────────────────────┼────────────────┤
  │ Tracklet association / tracklet fusion │ 视觉多目标跟踪、自动驾驶        │ ✓ 最接近       │
  ├────────────────────────────────────────┼─────────────────────────────────┼────────────────┤
  │ Track segment association              │ GMTI 雷达、运动目标检测         │ ✓ 最接近       │
  ├────────────────────────────────────────┼─────────────────────────────────┼────────────────┤
  │ Broken track repair / gap filling      │ 粒子物理、气象雷达              │ 类似           │
  ├────────────────────────────────────────┼─────────────────────────────────┼────────────────┤

  ├────────────────────────────────────────┼─────────────────────────────────┼────────────────┤

  关键发现：这个问题在学术界是分散研究的，没有统一的"碎片化航迹匹配融合"理论体系。但有几个高度相关的子领域。

  二、核心文献分析

  文献 1：Coraluppi & Carthel (2004) — "Recursive Track Fusion for Multi-Sensor Surveillance" (182 cited) ★★★★★

  来源：Information Fusion, ScienceDirect

  核心贡献：
  - 提出了 ATIF（Automatic Track Information Fusion） 框架
  - 定义了 track fragmentation ratio = 航迹数 / 目标数，量化碎片化程度
  - 提出递归融合方法：每收到新航迹段就更新融合状态，不需要等待所有数据
  - 关键公式：融合航迹的状态估计是局部航迹的加权平均，权重由协方差决定
  - 与我们的直接关联：他们明确讨论了"当局部传感器产生多条航迹段（tracklets）时，如何递归地将它们融合"——这正是我们要的段
  级融合框架

  对我们的启示：

  ▎ 递归融合 = 每收到一个新段就做一次融合更新，不需要等到所有段都收集完。这完美对应我们的"段聚合 + 桥接融合"思路。

  文献 2：Visina et al. (2020) — "Track-to-Track Fusion Using Inside Information From Local IMM Estimators" (19 cited)
  ★★★★★

  来源：JAIF, Bar-Shalom 团队

  核心贡献：
  - IMM 滤波器的航迹融合，考虑了模式条件估计（MCE）和模式概率
  - 提出了 GT2TFwXC（Gaussian T2TF with Crosscovariance） 方法
  - 关键发现：忽略 crosscovariance 会导致融合协方差过于乐观（ underestimated uncertainty）
  - 提出的方法性能接近集中式测量融合（CMF），但通信成本低

  对我们的启示：

  ▎ 我们的桥接融合器在做 A/B/C 三类区间融合时，必须考虑 crosscovariance。如果 R1 和 R2
  ▎ 的滤波器使用相同的目标运动模型（CV/CT），它们的估计误差是相关的，直接加权平均会低估不确定性。Bar-Shalom 的 GT2TFwXC
  ▎ 给出了计算方法。

  文献 3：Yuan et al. (2011) — "Heterogeneous Track-to-Track Fusion" (59 cited) ★★★★

  来源：IEEE InfoFusion, Bar-Shalom 团队

  核心贡献：
  - 处理异构传感器（主动雷达 + 被动雷达）的 T2TF
  - 两种方法：LMMSE 和 ML
  - 发现异构情况下 crosscovariance 既有正相关也有负相关，取决于传感器-目标几何关系
  - 无法解析计算 crosscovariance，需要用 Monte Carlo 估计

  对我们的启示：

  ▎ 我们的 R1 和 R2 虽然是同构传感器（都是天波超视距雷达），但它们的系统偏差符号相反（R1 偏差 +20km，R2 偏差
  ▎ -15km），这意味着它们的残余误差也是相关的。需要考虑 crosscovariance 的估计。

  文献 4：Qi et al. (2025) — "Multi-Radar Track Fusion Method Based on Parallel Track Fusion Model" (4 cited) ★★★

  来源：MDPI Electronics

  核心贡献：
  - 端到端深度学习融合方法（注意力机制 + LSTM 并行）
  - 通过不确定性加权动态调整融合权重
  - 实验表明融合精度比传统卡尔曼滤波低 79%

  对我们的启示：

  ▎ 这篇是深度学习方案，不适用于我们当前基于物理模型的框架。但其中提到的"不确定性加权动态调整融合权重"的思想可以借鉴——在
  ▎ B 类区间（仅一部雷达有数据），该雷达的协方差更大，融合时应赋予较低权重。

  文献 5：Shu et al. (2021) — "A Hybrid Tracking Algorithm for Multistatic Passive Radar" (24 cited) ★★★

  来源：IEEE Systems Journal

  核心贡献：
  - 构造两类航迹：高精度航迹 + 低精度航迹
  - 低精度航迹在满足条件时可转换为高精度航迹
  - 解决了低检测概率下航迹确认慢、连续性差的问题

  对我们的启示：

  ▎ 他们的"双精度航迹"概念与我们"段级融合"的思路异曲同工：不同质量的航迹段有不同的置信度，需要在融合时区分对待。

  五、结论与建议

  1. 段级架构（tracklet association + fusion）在文献中有坚实理论基础，Coraluppi 2004 的 ATIF 框架直接证明了可行性
  2. Bar-Shalom 的 GT2TFwXC 是融合模块的最佳选择，已有 IMM 扩展（Visina 2020）与我们当前的 3in1-imm 后端完美匹配
  3. crosscovariance 是关键难点——R1 和 R2 使用相同的运动模型，估计误差相关，忽略它会导致融合协方差被低估
  4. IEEE 5937272（GMTI 航迹段关联）是最直接相关的论文，84 次引用，需要下载
  5. 单目标作为多目标退化——文献中没有专门讨论这一点，但从我们的推导看，n_targets=1
