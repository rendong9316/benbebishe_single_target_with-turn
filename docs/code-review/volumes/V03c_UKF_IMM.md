## 2. 文件职责与接口契约

### 2.1 职责

IMM UKF 滤波器，封装 CV/CT 双模型的交互多模型滤波。每帧执行：模型混合（Mixing）→ 双模型独立 UKF 预测 → 组合输出 → 双模型独立 UKF 更新 → Pd-IPDA 似然度 → 贝叶斯概率更新 → 状态组合。对外暴露与 `ukf_jichu` 和 `ukf_zishiying` 一致的 `create/init/prepare/update` action 接口。[S][P]

### 2.2 输入输出与单位契约

| action | 输入 | 输出 |
|---|---|---|
| `create` | params, 雷达/Tx 位置, dt | IMM 模板（含 CV/CT 子滤波器 + 概率 + Markov 矩阵） |
| `init` | 模板, meas1, meas2 | 初始化两个子滤波器 |
| `prepare` | 模板 | `x_pred, P_pred, X_pred, z_pred, Z_pred, P_zz, imm`（7 输出） |
| `update` | 模板, PDA 加权新息 | `lon, lat, imm` |

`prepare` 返回中，tracker 实际使用 `x_pred`（CV 预测）、`z_pred`（CV 预测量测）、`P_zz`（CV 预测量测协方差）。组合预测 `x_pred_comb` 和 `z_pred_comb` 仅缓存，不返回给 tracker。[S][P]

## 3. 逐语句块审查

### 3.1 主调度器（第 37—51 行）

**语句职责。** `switch action` 分发到 4 个局部函数。[S]

**代码质量。** 与 `ukf_jichu` 风格一致。错误消息格式与 `ukf_jichu` 相同（无 identifier），遗留 `P3-01`。[S]

### 3.2 `create_imm` — 模板创建（第 58—117 行）

#### 3.2.1 CV/CT 子滤波器创建（第 67—80 行）

**语句职责。** 创建两个 `ukf_jichu` 实例，CT 覆盖 `model_type='CT'` 和 `turn_rate`。[S]

**数学正确性。** CT 默认转弯率 `1°/s = π/180 rad/s ≈ 0.01745 rad/s`，对应约 3.3 分钟完整圆周，合理。[M]

#### 3.2.2 Markov 转移矩阵（第 82—91 行）

**语句职责。** 从 `params` 读取转移概率，默认 `[0.9, 0.1; 0.1, 0.9]`。[S]

**数学正确性。** 行和为 1，是合法随机矩阵。稳态停留时间：CV 模型平均驻留 `1/0.005 = 200` 帧（`imm_Pi_CV_to_CT=0.005`），CT 模型平均驻留 `1/0.005 = 200` 帧。[M]

**问题记录。** `V03c-P2-01`：`imm_Pi_CV_to_CT=0.005` 对应转弯持续约 200 帧（6000 秒），远超典型转弯持续时间（通常 30-300 秒）。转弯概率过小可能导致 IMM 在转弯开始时 CT 概率上升缓慢，跟踪延迟。[M]

#### 3.2.3 Pd-IPDA 检测参数（第 101—105 行）

**语句职责。** 从 `params` 读取 `detection_probability` 和 `pda_pd_gate`，计算 `Pd_Pg` 和无检测似然。[S]

**数学正确性。** `L_no_det = 1 - Pd * Pg` 是 IPDA 无检测似然的简化形式。[M]

**问题记录。** `V03c-P1-01`：`Pd=1.0`（当前配置），`Pg=0.8647`（对应阈值 4），`Pd_Pg=0.8647`。但关联器实际门限为 72（`gate_sigma=6`），对应 `chi2cdf(72,2)≈1`。`Pg` 与真实门限不匹配，似然计算基于错误的检测概率。[S][M]

### 3.3 `init_imm` — 初始化（第 123—145 行）

**语句职责。** 初始化两个子滤波器，设置初始模型概率和缓存。[S]

**代码质量。** 两个子滤波器初始化逻辑完全重复（`Q_base`, `Q_ema`, `nis_history`），应提取为辅助函数。[S]

**问题记录。** `V03c-P2-02`：`imm.x = imm.ukf_cv.x` 和 `imm.P = imm.ukf_cv.P` 在初始化时只取 CV 模型，CT 模型未参与组合。若实际目标是 50/50 组合，应使用 `mu` 加权。[S]

### 3.4 `prepare_imm` — IMM 预测步（第 153—215 行）

#### 3.4.1 模型混合（第 160—182 行）

**语句职责。** 计算混合概率 `mu_mix`，混合初始状态和协方差。[S]

**数学正确性。**

```text
c_bar(j) = Σ_i Pi(i,j) * mu(i)     % 归一化常数
mu_mix(i,j) = Pi(i,j) * mu(i) / c_bar(j)
x_mix_j = Σ_i mu_mix(i,j) * x_i
P_mix_j = Σ_i mu_mix(i,j) * [P_i + (x_i - x_mix_j)(x_i - x_mix_j)']
```

标准 IMM 混合公式，包含模型均值离差外积项 `(dx*dx')`，正确。[M]

**数值稳定性。** `max(c_bar(j), 1e-12)` 防止除零。[S]

#### 3.4.2 双模型独立预测（第 184—188 行）

**语句职责。** 分别调用 `ukf_jichu('prepare', ...)`。[S]

**性能。** 2 × 9 = 18 次 `skywave_geometry` 调用（每个子滤波器 9 次）。[S]

#### 3.4.3 组合预测（第 190—203 行）

**语句职责。** 计算 `mu` 加权组合，但返回 CV 模型预测给 tracker。[S]

**数学正确性。**

```text
x_pred_comb = mu(1)*x_cv + mu(2)*x_ct
z_pred_comb = mu(1)*z_cv + mu(2)*z_ct
P_zz_comb = 0.5*P_zz_cv + 0.5*P_zz_ct    % ← 固定 0.5/0.5，不用 mu
```

**问题记录。** `V03c-P1-02`：`P_zz_comb` 固定使用 0.5/0.5 加权，而非当前模型概率 `mu`。这导致组合测量协方差在 CT 占优时仍然反映 CV 的特性，低估或高估实际不确定性。此外，`P_zz_comb` 未加入模型均值离差 `Σ mu_i * (z_i - z_comb)(z_i - z_comb)'`，这在 IMM 理论中是必须的。[M]

**问题记录。** `V03c-P1-03`：返回给 tracker 的是 CV 模型的 `x_pred, z_pred, P_zz`（第 196-198 行），而非组合预测。即使 CT 概率接近 1，波门中心仍然是 CV 预测。这导致转弯时 CT 的优势无法传递到关联层。[S][M]

#### 3.4.4 缓存（第 205—214 行）

**语句职责。** 保存所有中间量供 `update_imm` 使用。[S]

**代码质量。** 缓存字段完整，但 `P_zz_comb` 的计算质量存疑（见 V03c-P1-02）。[S]

### 3.5 `update_imm` — IMM 更新步（第 222—295 行）

#### 3.5.1 纯预测帧（第 227—233 行）

**语句职责。** 无新息时保留预测状态，似然设为 `L_no_det`。[S]

**数学正确性。** 纯预测帧的似然度等于无检测似然，合理。[M]

#### 3.5.2 量测重建与新息计算（第 235—246 行）

**语句职责。** 从 PDA 加权新息重建量测 `z_weighted = innov_w + z_pred_cv`，再计算各模型新息。[S]

**数学正确性。** `innov_cv = z_weighted - z_pred_cv` 即 PDA 加权新息本身。`innov_ct = z_weighted - z_pred_ct` 是用 CV 量测减去 CT 预测量测，方向可能完全不同。[M]

**方位角包裹。** `round(innov(2)/360)` 处理跨越 0°/360° 的情况，正确。[M]

#### 3.5.3 各模型独立更新（第 248—250 行）

**语句职责。** 委托 `ukf_jichu('update', ...)`。[S]

**性能。** 2 次 `ukf_jichu('update')`，每次内部无额外 `skywave_geometry` 调用（使用缓存）。[S]

#### 3.5.4 NIS 记录（第 252—261 行）

**语句职责。** 记录各模型 NIS 到 `nis_history`。[S]

**数学正确性。** `nis = innov' * P_zz \ innov`，使用缓存的 `P_zz`。[M]

**问题记录。** `V03c-P2-03`：`imm.nis_history = imm.ukf_cv.nis_history` 仅镜像 CV 模型的 NIS，CT 模型的 NIS 未同步。若外部诊断代码读取 `imm.nis_history`，只能看到 CV 模型的统计。[S]

#### 3.5.5 Pd-IPDA 似然度（第 263—268 行）

**语句职责。** 计算各模型的检测似然。[S]

**数学正确性。**

```text
L_i = Pd_Pg * (2π)^(-nz/2) * |P_zz_i|^(-1/2) * exp(-0.5 * nis_i)
```

标准 IPDA 似然形式。`max(det(...), 1e-30)` 防止行列式非正。[M]

**问题记录。** `V03c-P1-04`：似然中使用 `nz=3`（量测维数），但 `P_zz_cv` 和 `P_zz_ct` 是 3×3 矩阵（含多普勒）。而 `PDA` 关联使用的是 2D 新息和 2D `P_zz(1:2,1:2)`。似然与关联使用不同维度的统计，结果不可直接比较。[S][M]

#### 3.5.6 贝叶斯概率更新（第 270—278 行）

**语句职责。** 组合似然与先验 Markov 概率，钳位到 `[0.02, 0.95]`，归一化。[S]

**数学正确性。**

```text
mu_new(i) ∝ L_i * c_bar(i)    % c_bar = Pi' * mu (混合概率)
mu_new = clamp(mu_new, [0.02, 0.95])
mu_new = mu_new / sum(mu_new)
```

标准 IMM 概率更新。钳位防止概率过早收敛到 0 或 1。[M]

#### 3.5.7 状态组合（第 280—288 行）

**语句职责。** 组合状态和协方差。[S]

**数学正确性。**

```text
x_comb = mu(1)*x_cv + mu(2)*x_ct
P_comb = mu(1)*P_cv + mu(2)*P_ct    % ← 缺少模型均值离差
```

**问题记录。** `V03c-P1-05`：顶层组合协方差 `imm.P` 仅使用模型协方差的加权平均，缺少各模型均值相对组合均值的外积项 `Σ mu_i * (x_i - x_comb)(x_i - x_comb)'`。这导致 `P_comb` 在模型分歧大时（如转弯开始）严重低估不确定性。[M]

### 3.6 `keep_prediction`（第 301—310 行）

**语句职责。** 纯预测帧保留缓存状态。[S]

**代码质量。** 简洁正确。[S]

## 5. 文件级结论

- **数学可信度**：IMM 混合公式正确（含离差项），但组合测量协方差固定 0.5/0.5 且缺少模型均值离差，组合状态协方差缺少离差项，导致不确定性被低估。门中心固定 CV 预测而非组合预测，转弯时 CT 优势无法传递到关联层。似然计算使用 3D 量测统计而关联使用 2D，维度不一致。
- **代码质量**：两个子滤波器初始化逻辑重复；`trimf_val` 函数重复实现；CT 模型缺少自适应 Q。
- **性能**：每帧 18 次 `skywave_geometry` 调用（2 模型 × 9 Sigma 点），是 MC 循环中的主要热点。
- **测试充分性**：端到端冒烟通过，但 IMM 概率收敛、组合协方差正定性、似然计算等关键行为未获独立验证。
- **剩余未验证项**：不同 `imm_Pi` 值对转弯 RMSE 的影响；组合协方差缺离差项对融合的影响；门中心固定 CV 对转弯关联率的影响。
