## 3. 逐语句块审查

### 3.1 快速查找映射（第 63—68 行）

### 3.2 BC 互协方差初始化（第 75—81 行）

**语句职责。** 若使用 BC，为每对航迹初始化 4×4 零矩阵和 `has_prev` 标志。[S]

**数学正确性。** 首帧无前驱，互协方差为零（独立假设）。[M]

### 3.3 主循环 — 双源融合（第 136—228 行）

#### 3.3.1 SCC（第 147—150 行）

**语句职责。** 调用 `fuse_scc`，等效权重 0.5。[S]

**数学正确性。** SCC 假设独立，直接信息矩阵相加。[M]

**问题记录。** `V06-P2-01`：`fused_trk.w = 0.5` 硬编码。SCC 在数学上等价于 CI 在 `w=0.5` 时的特例，但文档注释说"隐含 w=0.5"，而 `fuse_scc` 的实现是直接相加 `inv(P1)+inv(P2)`，不经过 CI 的权重优化。这两者在数学上等价（当 P1=P2 时 CI 的 `w_opt=0.5`），但语义不同。[S]

#### 3.3.2 CI（第 152—155 行）

**语句职责。** 调用 `fuse_ci` 优化 `w`。[S]

**数学正确性。** `fuse_ci` 使用 `fminbnd` 最小化 `det(P_fused)`。[M]

**性能。** `fminbnd` 每次融合需多次 `det(P_inv)` 计算，每次涉及 4×4 矩阵求逆。在 MC 循环中此开销不可忽视。[S]

#### 3.3.3 FCI（第 157—160 行）

**语句职责。** 调用 `fuse_fci` 解析式计算 `w`。[S]

**数学正确性。** `w_fci = tr(P1)^{-1} / (tr(P1)^{-1} + tr(P2)^{-1})`。[M]

**问题记录。** `V06-P1-01`：FCI 用迹作为不确定度代理，但 `P1` 和 `P2` 的元素量纲不同（deg², deg²/s² 等），迹混合了不同物理量的单位。迹的大小受状态分量单位选择影响，因此 `w_fci` 的权重分配不是单位不变的。[M]

#### 3.3.4 BC 互协方差传播（第 162—224 行）

**语句职责。** BC 方法的 P12 预测、迹收缩、稳定性约束。[S]

**数学正确性。**

```matlab
% 预测步
P12_pred = F_cv_dt * P12_cell{p} * F_cv_dt' + Q_half;
Q_half = trk1.ukf.Q * 0.5;

% 更新步（迹收缩）
alpha = max(0.1, min(1.0, alpha1 * alpha2));
alpha1 = sqrt(trace(P1) / trace(P1_pred));
alpha2 = sqrt(trace(P2) / trace(P2_pred));
P12_new = alpha * P12_pred;

% 约束
max_p12 = 0.8 * min(diag(P1)) * eye(4);
```

**问题记录。** `V06-P1-02`：`Q_half = trk1.ukf.Q * 0.5` 仅使用 R1 的 Q。R1 和 R2 的 Q 可能不同（R1 `scale=1e5`，R2 `scale=2e5`），但 P12 传播中只计入 R1 的一半 Q。这在物理上不对称。[M]

**问题记录。** `V06-P1-03`：迹收缩比 `alpha = sqrt(trace(P1)/trace(P1_pred)) * sqrt(trace(P2)/trace(P2_pred))` 是对 `(I-K1*H1)*(I-K2*H2)'` 的近似。但 `P1` 和 `P2` 的量纲混合（deg² 与 deg²/s²），迹本身没有明确的物理意义，迹收缩比作为 `P12` 的标量因子缺乏严格的数学推导。[M]

**问题记录。** `V06-P1-04`：稳定性约束 `max_p12 = 0.8 * min(diag(P1)) * eye(4)` 使用 `min(diag(P1))` 统一限制所有对角元（包括速度分量）。但位置和速度量纲不同，用位置的最小方差约束速度协方差在物理上不合理。[M]

**问题记录。** `V06-P2-02`：约束循环中对第 `ii` 行/列等比例缩放，但缩放后 `P12_new` 可能不再对称（第 209 行缩放行，第 210 行缩放列，如果 `scale` 不同会导致不对称）。当前代码中 `scale = sqrt(...)` 对行和列使用相同的缩放因子，因此对称性得以保持。但代码结构易误导未来维护者。[S]

### 3.4 单源透传（第 242—269 行）

**语句职责。** 仅 R1 或仅 R2 有数据时直接透传，BC 模式下重置 `has_prev`。[S]

**数学正确性。** 单源透传是合理的退化策略。[M]

**问题记录。** `V06-P2-03`：BC 模式下单源数据导致 `has_prev(p) = false`，下一帧需要从零重新开始 P12。这可能导致融合结果在单源帧后出现协方差跳变。[S]

### 3.5 `find_track` 辅助函数（第 294—305 行）

**语句职责。** 线性扫描 `trackList` 查找航迹。[S]

**数学正确性。** 无数学运算。[S]

**代码质量。** 航迹数量通常很小（1-5），线性扫描 `O(n)` 可接受。[S]

## 5. 文件级结论

- **数学可信度**：BC 融合公式（`S=P1+P2-P12-P12'`，`K=(P1-P12)/S`）正确，但 P12 的经验传播（迹收缩比 + 统一对角约束）缺乏严格的数学推导。FCI 迹权重在量纲混合状态下不是单位不变的。
- **代码质量**：硬编码 SCC 权重 0.5；约束代码可读性差；BC 单源重置导致协方差跳变。
- **性能**：CI 的 `fminbnd` 优化在 MC 循环中是主要热点；FCI 解析式计算 `O(1)`。
- **测试充分性**：端到端冒烟通过，但 BC P12 的联合协方差半正定性、FCI 迹权重在不同量纲下的行为未获独立验证。
- **剩余未验证项**：BC 经验参数敏感性分析；FCI 迹权重在 R1/R2 精度差异大时的表现。
