## 3. 逐语句块审查

### 3.1 调度函数（第 39—66 行）

**语句职责。** `switch upper(method)` 分发到四个局部函数。[S]

**代码质量。** BC 通过 `varargin` 传递 `P12`，其他算法不需要。接口一致性好。[S]

### 3.2 `fuse_scc`（第 111—138 行）

#### 3.2.1 正则化（第 115—116 行）

**语句职责。** 对输入协方差进行正则化。[S]

#### 3.2.2 信息矩阵加法（第 123—124 行）

**语句职责。** `P_fused_inv = inv(P1) + inv(P2)`。[S]

**数学正确性。**

```text
P_fused = (P1^{-1} + P2^{-1})^{-1}
```

标准独立估计的信息融合。[M]

**性能。** 两次 `inv` + 一次矩阵加法 + 一次 `inv`。4×4 矩阵求逆开销小，但 MC 循环中重复执行。[S]

**问题记录。** `V06b-P2-01`：使用 `inv()` 显式求逆而非反斜杠 `\`。`inv(P1) + inv(P2)` 可改写为 `chol(P1)\(chol(P2)\...` 或使用 Schur 补，数值稳定性更好。[S]

#### 3.2.3 状态融合（第 132 行）

**语句职责。** `x_fused = P_fused * (P1\x1 + P2\x2)`。[S]

**数学正确性。** 使用反斜杠求解线性系统，比显式 `inv(P1)*x1` 数值稳定。[M]

### 3.3 `fuse_bc`（第 186—231 行）

#### 3.3.1 正则化与 P12 处理（第 191—200 行）

**语句职责。** 正则化输入，处理缺失 `P12`。[S]

#### 3.3.2 新息协方差 S（第 208 行）

**语句职责。** `S = P1 + P2 - P12 - P12'`。[S]

**数学正确性。** 标准 BC 公式。当 `P12=0` 时退化为 `P1+P2`（独立假设）。[M]

#### 3.3.3 融合增益（第 215 行）

**语句职责。** `K_bc = (P1 - P12) / S`。[S]

**数学正确性。** 标准 BC 增益。[M]

#### 3.3.4 融合状态（第 222 行）

**语句职责。** `x_fused = x1 + K_bc * (x2 - x1)`。[S]

**数学正确性。** 对新息的加权修正。[M]

#### 3.3.5 融合协方差（第 229 行）

**语句职责。** `P_fused = P1 - K_bc * (P1 - P12')`。[S]

**数学正确性。** 标准 BC 协方差更新。[M]

**问题记录。** `V06b-P2-02`：BC 公式中 `P12'`（转置）的使用容易出错。如果 `P12` 不对称（经验传播可能导致不对称），`P1 - P12'` 与 `P1 - P12` 不同，结果依赖于 `P12` 的对称性。`run_track_fusion.m:209-210` 的行列缩放虽使用相同因子保持对称，但任何数值误差都可能导致微小不对称。[S][M]

### 3.4 `fuse_ci`（第 273—317 行）

#### 3.4.1 预计算逆矩阵（第 283—284 行）

**语句职责。** 预计算 `P1_inv` 和 `P2_inv`，避免优化循环中重复求逆。[S]

**数学正确性。** CI 优化只改变权重 `w`，不改变 `P1_inv` 和 `P2_inv`。预计算合理。[M]

**问题记录。** `V06b-P2-03`：`inv(P1)` 和 `inv(P2)` 在 `regularize_cov` 之后计算，但 `ci_cost` 函数内部直接使用预计算的逆矩阵。如果 `regularize_cov` 改变了 `P1` 的特征向量方向（仅改变特征值大小），逆矩阵的方向不变。正确。[S]

#### 3.4.2 优化（第 292—297 行）

**语句职责。** `fminbnd` 最小化 `det(P_fused)`。[S]

**数学正确性。** `cost(w) = 1/det(w*P1_inv + (1-w)*P2_inv)`。`det` 越小，融合协方差越紧凑。[M]

**性能。** `fminbnd` 默认收敛需 20-50 次目标函数评估。每次评估涉及 4×4 矩阵求逆 + `det`。在 MC 循环中，此开销可能占融合总耗时的 50% 以上。[S]

#### 3.4.3 信息融合（第 303—311 行）

**语句职责。** 用最优 `w` 执行信息矩阵凸组合。[S]

**数学正确性。** 标准 CI 公式。[M]

### 3.5 `ci_cost`（第 333—340 行）

**语句职责。** 计算给定 `w` 下的融合协方差行列式。[S]

**数学正确性。** `cost = 1/det(P_inv)` 等价于 `det(P_fused)`。[M]

**问题记录。** `V06b-P2-04`：`1/det(P_inv)` 在 `det(P_inv)` 接近零时可能溢出。虽然 4×4 正定矩阵的行列式通常远离零，但在极端情况下（如 `P1≈P2` 且 `w≈0.5` 时 `P_inv` 接近奇异），`det` 可能数值不稳定。[S]

### 3.6 `fuse_fci`（第 382—421 行）

#### 3.6.1 迹权重（第 394—396 行）

**语句职责。** `w_fci = tr(P1)^{-1} / (tr(P1)^{-1} + tr(P2)^{-1})`。[S]

**数学正确性。** 迹越小（总不确定度越小）的传感器获得更大权重。[M]

**问题记录。** `V06b-P1-01`：迹 `tr(P)` 混合了不同量纲的对角元（deg² + deg²/s² + deg² + deg²/s²）。迹的值依赖于状态分量的单位选择。例如，将速度从 deg/s 改为 rad/s，`tr(P)` 会改变约 `(206265)^2 ≈ 4.25×10^{10}` 倍，导致 `w_fci` 完全偏向位置分量。这不是单位不变的权重分配。[M]

#### 3.6.2 信息融合（第 408—415 行）

**语句职责。** 与 CI 相同的信息形式融合。[S]

**数学正确性。** 标准公式。[M]

## 5. 文件级结论

- **数学可信度**：SCC/BC/CI 公式实现正确。FCI 迹权重在量纲混合状态下不是单位不变的，这是 FCI 方法的固有缺陷（非本文件特有）。
- **代码质量**：SCC 使用 `inv()` 而非反斜杠；BC 的 `P12'` 使用在 `P12` 不对称时行为不确定；`ci_cost` 在极端情况下可能溢出。
- **性能**：CI 的 `fminbnd` 优化是融合模块的主要热点，每次融合约 20-50 次矩阵求逆。FCI 解析式计算 `O(1)`。
- **测试充分性**：端到端冒烟通过，但 CI 优化收敛性、FCI 迹权重在不同单位下的行为未获独立验证。
- **剩余未验证项**：CI 权重 `w_opt` 在不同 `P1/P2` 比例下的收敛行为；FCI 迹权重在 R1/R2 精度差异极大的场景下的表现。
