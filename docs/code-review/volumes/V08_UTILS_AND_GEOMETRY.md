## 2. 文件职责与接口契约

### 2.1 职责

该文件以 action dispatcher 形式集中提供球形地球、固定电离层虚高假设下的天波双基地几何量：常量、地心角、地表弦长、单段天波斜距、双基地群距离、球面初始方位角、群距离变化率及组合几何量。[S]

仓库调用点表明，`simulation/generate_frame_detections.m` 用它生成群距离、方位和多普勒真值，`ukf/ukf_jichu.m` 用同一函数构造 UKF 量测模型；多个单目标、多目标、蒙特卡洛入口以及反解、跟踪模块也调用 `group_range`。因此该文件是仿真端与滤波端共享的核心模型边界，而不是普通的低影响工具函数。[P]

### 2.2 输入输出与单位契约

| action | 输入 | 输出 | 源码可推定单位 |
|---|---|---|---|
| `constants` | 无附加参数 | `R_e`, `H` | m, m |
| `geocentric_angle` | `lon1, lat1, lon2, lat2` | `sigma` | 输入 deg，输出 rad |
| `chord_length` | 两点经纬度 | `D` | m |
| `slant_range` | `D` | `r` | 输入/输出 m |
| `group_range` | Tx、Rx、目标经纬度 | `Rg` | m |
| `doppler` | Tx、Rx、目标经纬度及目标经纬度变化率 | `vd` | 经纬度 deg、变化率 deg/s，输出 m/s |
| `azimuth` | 起点与终点经纬度 | `az` | 输入 deg，输出 deg，归一到 `[0,360)` |
| `full` | Tx、Rx、目标经纬度 | 8 个几何量 | `r/D` 为 m，`sigma` 为 rad，`az` 为 deg |

上述单位主要写在公式和调用语义中，函数没有显式类型、尺寸、有限性或单位验证。[S][P]

### 2.3 模型边界

`R_e=6371000 m`、`H=300000 m` 均为固定常量；地球按球体处理，电离层按固定虚高处理。单段公式

```text
r = sqrt(D^2 + (2H)^2)
```

可改写为 `2*sqrt((D/2)^2+H^2)`，对应在弦长中点上方高度 `H` 的对称两腿路径近似。[M] 文件头把它称为“电离层双跳”，术语容易误导：从单段端点到端点看，它更像一次电离层反射形成的两腿传播；整个双基地量测则由 Tx→目标和目标→Rx 两个这种单段相加。建议在后续文档或代码注释中明确“跳数”的定义。[S][M]

## 3. 逐语句块审查

### 3.1 文件头与公开 action 清单（第 1—30 行）

**语句职责。** 第 1—9 行说明模块用途及仿真端/UKF 端共享目标；第 10—20 行列出核心公式和固定常量；第 21—29 行枚举 8 个公开 action 及返回顺序；第 30 行结束分隔。[S]

**数学与单位。** 地心角、弦长、斜距、群距离的公式与后续实现一致。`D=2R_e sin(sigma/2)` 是球内弦长，不是地表大圆弧长；`Rg=r_tx+r_rx` 的输出是总传播路径长度。[M] 第 15 行把 `v_d` 定义为总路径变化率，单位应为 m/s；它是“多普勒速度/距离率”，不是 Hz 域的多普勒频移。若下游需要频移，仍需结合载频或波长换算。[S][M]

**代码质量。** 公共 action 清单完整且与 switch 分支一致；`full` 的 8 输出顺序也被明确记录。[S] 但文件头没有说明：输入必须为标量、经纬度与变化率单位、action 大小写敏感、重合点/对跖点方位未定义、非法数值处理策略。当前注释容易让调用者误以为接口可直接向量化。[S]

**复杂度。** 仅注释，无运行复杂度。

**建议。** 在接口注释中补齐“标量 double、经纬度 deg、变化率 deg/s、距离 m、距离率 m/s”；将“多普勒”明确写为“群距离率”，并说明固定球体/固定虚高是建模近似。[S][M]

### 3.2 主函数签名、常量与 dispatcher（第 32—41、93—96 行）

**语句职责。** 第 32 行以 `varargout` 和 `varargin` 实现多入口函数；第 34—35 行在每次调用时建立固定常量；第 37 行按 `action` 分发；第 38—40 行返回常量；第 93—95 行对未知 action 抛错；第 96 行结束主函数。[S]

**数学与单位。** `R_e=6371000.0`、`H=300000.0` 的单位由文件头定义为米。常量数值在主函数内唯一声明，并向 `doppler_impl` 显式传递，避免局部函数依赖隐式全局状态。[S]

**代码质量。** 函数式 dispatcher 符合项目禁止 OOP 的约束，并把共享常量控制在单文件内。[S] 但 action 是大小写敏感的裸字符串协议；错误标识实际为格式化消息文本 `skywave_geometry: unknown action ...`，并未使用 MATLAB 的 `error(identifier,message,...)` 形式，因此调用者不能稳定按异常 identifier 捕获。[S]

`constants` 分支总是给 `varargout{1:2}` 赋值，但没有检查调用者请求的输出数；其他 action 也没有声明各自允许的 `nargout`。接口错误只能由 MATLAB 在更晚处以通用错误暴露。[S]

**复杂度。** action 分支数固定，调度时间和额外空间均为 `O(1)`。[S]

**建议。** 使用 `validatestring` 或统一的 `lower(char(action))` 规范 action；采用稳定异常标识，例如 `error('skywave_geometry:UnknownAction',...)`；按 action 校验 `nargin/nargout`。常量可保留在本函数中，也可由私有常量辅助函数集中返回，避免未来多个分支分别演化。[S]

### 3.3 参数拆包与输入校验（第 42—43、46—47、51—52、55—56、65—66、70—71、74—75 行）

**语句职责。** 各分支通过 `deal(varargin{:})` 或直接索引 `varargin{1}` 拆包输入。[S]

**数学与单位。** 源码不检查经度、纬度、距离、变化率的数值类型、实数性、标量性、有限性和单位；也不限制纬度到 `[-90,90]`，不处理 `NaN/Inf`，不拒绝负弦长 `D`。[S] 经纬度超出常规范围有时仍会因三角函数周期性得到数值结果，但那不等于输入语义有效；负 `D` 在平方后会静默产生与正 `D` 相同的斜距。[M]

**代码质量。** 参数个数错误会在 `deal` 或 cell 索引处报通用错误，无法指出具体 action 的契约。更重要的是，所有公式使用标量运算符 `^`、`*`、`/`，所以该实现实际上只可靠支持标量；若传入同形向量，`sin(...)^2`、`cos(...)*cos(...)` 等会触发矩阵幂/矩阵乘法错误或产生非预期矩阵运算。[S]

**复杂度。** 标量输入为 `O(1)`；当前没有定义批量输入复杂度，因为接口未实现逐元素向量化。[S]

**建议。** 每个 action 明确执行 `nargin`、`nargout` 和 `validateattributes`；经纬度至少要求 `real finite scalar`，纬度要求 `[-90,90]`，`D` 要求 `nonnegative`，变化率注明 deg/s。若确需批量能力，应系统性改为 `.^`、`.*`、`./` 并定义广播/尺寸一致性规则，而不是只局部替换。[S][M]

**问题记录。** `V08-P2-01`：公开接口无参数个数、类型、范围、有限性和标量性校验，错误定位差，非法输入可能静默通过或在底层以非领域错误失败。[S]

### 3.4 地心角分支与 Haversine 辅助函数（第 42—44、99—110 行）

**语句职责。** 第 42—44 行调用 `geocentric_angle_impl`。第 103—106 行把经纬差和纬度转换为弧度；第 107 行计算 Haversine 中间量 `a`；第 108 行把 `a` 钳位到 `[0,1]`；第 109 行用稳定的 `atan2` 形式求 `sigma`；第 110 行结束函数。[S]

**数学与单位。** 实现对应

```text
a = sin²(Δφ/2) + cosφ1 cosφ2 sin²(Δλ/2)
sigma = 2 atan2(sqrt(a), sqrt(1-a))
```

输入为度，`deg2rad` 后参与三角运算，输出 `sigma` 为弧度。[M] 第 108 行可抑制浮点舍入导致的 `a<0` 或 `a>1`，避免平方根产生复数/NaN，是必要的数值防护。[S][M]

**边界。** 重合点得到 `sigma=0`；对跖点理论上得到 `sigma=pi`。Haversine 在小角度距离上比直接余弦定理稳定。[M] 但 `NaN/Inf` 不会被转换为有意义的领域错误；对超范围纬度也不拒绝。[S]

**代码质量。** 对标量实现简洁、公式可读。第 107 行混合使用 `^` 与 `*` 明确限制了向量输入；函数名未显式表达输入为度、输出为弧度。[S]

**复杂度。** 固定次数三角函数，时间 `O(1)`、空间 `O(1)`。[S]

**建议。** 保留钳位；补充标量/有限性/纬度范围校验。若未来批量化，使用逐元素运算并增加同尺寸测试。可考虑命名或注释强化 `_deg`/`_rad` 单位边界。[S][M]

### 3.5 弦长计算（第 46—49、57—60、76—79、153—156 行）

**语句职责。** `chord_length` 分支先求地心角，再计算 `2R_e sin(sigma/2)`；`group_range`、`full` 和 `doppler_impl` 各自重复相同的 Tx/Rx 弦长计算。[S]

**数学与单位。** 对半径为 `R_e` 的球，两点球心夹角为 `sigma` 时，弦长

```text
D = 2 R_e sin(sigma/2)
```

量纲为米，且理论范围为 `[0,2R_e]`。[M] 该量是穿过球体内部的直线弦，不是地表传播距离 `R_e*sigma`。模型随后把它作为对称虚高路径的水平基线，这是项目的建模选择，不应与真实电离层射线追踪混同。[M]

**代码质量。** 公式本身正确，但同一计算在多个 action 和 `doppler_impl` 中重复，增加未来修改不一致的风险。[S] 由于使用标量 `sin` 和普通乘法，对标量清晰且开销小。

**复杂度。** 每次弦长为 `O(1)`；`group_range/full/doppler` 都计算两次。[S]

**建议。** 提取单段几何辅助函数，一次返回 `sigma,D,r,az`；`group_range`、`full`、`doppler` 复用。这样既减少重复，也能让模型公式在一个位置维护。[S]

### 3.6 斜距计算（第 51—53、61—62、80—81、157—158 行）

**语句职责。** `slant_range` 对输入 `D` 计算单段路径；其余三个流程分别对 Tx 和 Rx 弦长重复同一公式。[S]

**数学与单位。** 实现

```text
r = sqrt(D² + (2H)²) = 2 sqrt((D/2)² + H²)
```

若 `D,H` 均为米，则 `r` 为米。[M] 当 `D=0` 时返回 `2H=600 km`，意味着即使地面端点重合，模型仍保留上行加下行的固定虚高路径；这是该虚高模型的直接结果，不是普通欧氏点间距离。[M] 对有效 `D>=0`，`dr/dD=D/r`，斜距随弦长单调增加。[M]

**代码质量。** 公式清楚，但 `D^2` 限制标量输入；负 `D` 被平方掩盖；复数或非有限输入未拒绝。[S] 固定 `H` 不能按场景或雷达链路配置，也无法表达 Tx/目标和目标/Rx 两段不同虚高。[S]

**复杂度。** 时间/空间均 `O(1)`。[S]

**建议。** 校验 `D` 为有限非负标量；若项目需要研究电离层敏感性，应把 `H` 作为显式配置或增加可选 action 参数，同时保证仿真端和滤波端使用同一配置。[S][P]

**问题记录。** `V08-P2-02`：固定 `H` 与固定 `R_e` 无配置注入点，限制场景化建模和敏感性分析；修改常量必须改源码。[S]

### 3.7 群距离与 `full` 组合输出（第 55—63、74—91 行）

**语句职责。** `group_range` 分别计算 Tx→目标和 Rx→目标的 `sigma,D,r` 后返回 `r_tx+r_rx`；`full` 重做相同距离计算，并增加两站到目标的方位角，按固定顺序返回 8 个量。[S]

**数学与单位。** `Rg=r_tx+r_rx` 是两段路径之和，单位米。[M] Tx/Rx 顺序对群距离和两段距离集合具有交换对称性，但 `full` 的输出槽位有方向语义，交换站点会交换对应输出而不是保持每个槽位不变。[M]

**代码质量。** `full` 对诊断和复用有价值，但裸 `varargout{1..8}` 缺少结构化字段名，调用者必须严格记忆顺序；少接输出会丢弃后项，多请求输出则可能触发未赋值错误。[S] `group_range` 与 `full` 重复 7 行核心几何，`doppler_impl` 又重复一次。[S]

**复杂度。** `group_range` 为固定两段计算，`O(1)`；`full` 额外两次方位角计算，仍为 `O(1)`。[S]

**建议。** 内部建立 `segment_geometry` 辅助函数；若要降低输出顺序风险，可增加返回结构体的 action，同时保留现有 action 兼容调用方。不要在没有迁移调用点的情况下直接改变 `full` 输出顺序。[S][P]

### 3.8 方位角分支与 `azimuth_impl`（第 70—72、82—83、113—124、159—160 行）

**语句职责。** `azimuth` action 调用局部函数；第 117—119 行将经差和两纬度转为弧度；第 120—122 行计算大圆初始切向的 `y,x`；第 123 行经 `atan2` 转成度并归一到 `[0,360)`；`full` 和 `doppler_impl` 复用该函数求“站点→目标”的初始方位。[S]

**数学与单位。** 公式是球面两点间从起点出发的大圆**初始方位角**：

```text
y = sinΔλ cosφ2
x = cosφ1 sinφ2 - sinφ1 cosφ2 cosΔλ
az = mod(atan2(y,x)·180/pi, 360)
```

输入、输出均为度。[M]

**边界。** 两点重合时 `x=y=0`，MATLAB 通常给出数值 0，但数学方位未定义；对跖点也没有唯一大圆方向；极点附近 `x,y` 可能同时很小，方位数值敏感。[M] 当前实现没有检测或标注这些退化情况。[S]

**代码质量。** 对一般标量点对公式简洁；函数名和文件头已说明是球面方位角，但没有强调它是“起点处初始方位”，这对后续多普勒投影非常关键。[S][M]

**复杂度。** `O(1)` 时间、`O(1)` 空间。[S]

**建议。** 对重合点、近对跖点定义明确策略：返回 `NaN`、抛错或由调用方显式接受；在注释中强调初始方位只属于起点局部切平面，不能直接当作终点处速度投影方向。[M]

**问题记录。** `V08-P2-03`：重合点、对跖点和极点附近的方位退化没有显式处理，当前可能返回看似有效但无物理定义的角度。[S][M]

### 3.9 经纬度变化率到局部 ENU 速度（第 143—150 行）

**语句职责。** 第 143—144 行声明多普勒辅助函数及参数；第 146 行把目标纬度转为弧度；第 148—150 行把目标的经纬度变化率换算为目标位置局部东、北速度。[S]

**数学与单位。** 对半径 `R_e` 的球面，若 `lon_rate=dλ_deg/dt`、`lat_rate=dφ_deg/dt`，则

```text
v_east  = R_e cosφ · dλ_rad/dt
v_north = R_e · dφ_rad/dt
```

代码通过乘 `pi/180` 完成 deg/s→rad/s，结果为 m/s，公式在球形地球局部坐标下正确。[M]

**边界。** 在极点 `cosφ=0`，经度坐标本身退化；代码会把有限经度变化率映射为零东向速度，但此处经度变化率并不是稳定的物理参数。[M] 对超范围纬度或异常大变化率没有校验。[S]

**代码质量。** 换算集中、可读，单位关系可从注释推定，但输入命名没有 `_deg_s` 后缀。[S]

**复杂度。** `O(1)`。[S]

**建议。** 明确变化率单位并验证目标纬度；若轨迹以后以 ECEF/ENU 速度为主，建议直接允许传入局部速度，避免在极区或跨经度边界时依赖退化的经纬度率表示。[M]

### 3.10 多普勒主流程（第 65—68、127—168 行）

**语句职责。** `doppler` action 拆包 8 个业务参数，并把 `R_e,H` 传给 `doppler_impl`。辅助函数先换算目标 ENU 速度，再为 Tx、Rx 两段计算地心角、弦长、斜距、站点到目标的初始方位，调用 `path_rate_impl` 得到两段路径变化率，最后求和。[S]

**数学与单位。** 目标输出按定义是

```text
vd = d(r_tx+r_rx)/dt = dr_tx/dt + dr_rx/dt
```

若每段方向导数正确，输出单位为 m/s。[M] 该值的正号表示总路径增长、负号表示总路径缩短；源码和文件头没有明确陈述符号约定，但公式隐含了这一点。[S][M]

**关键正确性复核。** 第 159—160 行取得的是“站点处、指向目标”的大圆初始方位角 `azimuth_impl(station,target)`；第 163—164 行却把这个角用于投影**目标位置处**定义的 `v_east,v_north`。[S] 在球面上，一条大圆沿途相对于各点局部北向的方位通常会变化。目标端距离梯度应使用目标局部切平面中“远离站点”的方向，可由目标→站点的初始方位反向得到：

```text
az_out_at_target = mod(azimuth_impl(tgt_lon,tgt_lat,station_lon,station_lat)+180,360)
```

而不是直接使用站点端的初始方位。[M] 只有同一经线、赤道路径或距离较短等特殊情形，两者才可能相同或近似相同。因此当前实现对一般长距离、非特殊大圆路径的 `dσ/dt` 投影方向存在系统误差；OTH 场景正是长距离应用，该误差不能仅凭“仿真端与 UKF 端共用同一函数”消除。共用函数只消除了生成模型与滤波模型之间的失配，不证明模型相对真实球面运动正确。[M][P]

**代码质量。** 主流程分步注释清楚，且仿真端和 UKF 端确实复用同一 action。[P] 但几何量与 `full` 分支重复计算；命名 `doppler` 容易与 Hz 混淆；未声明符号约定和速度参考点。[S]

**复杂度。** 固定两段：2 次 Haversine、2 次斜距、2 次方位、2 次路径率，时间和空间均 `O(1)`。[S]

**建议。** 优先用解析梯度或有限差分对球面距离率做独立验证；将每段投影方向改为目标端的路径梯度方向，并建立至少以下测试：纯北/纯东、赤道、同经线、一般斜向长距离、数值中心差分对照。修复时必须同步检查 `generate_frame_detections` 与 `ukf_jichu` 的符号期望，但继续保持两端共用同一实现。[M][P]

**问题记录。** `V08-P1-01`：多普勒把站点端初始方位用于目标端 ENU 速度投影；一般球面大圆上两端局部方位不同，导致群距离率方向导数存在系统误差。[S][M]

### 3.11 单段路径率辅助函数（第 171—179 行）

**语句职责。** 第 175 行声明 `path_rate_impl`；第 176 行把方位转成弧度；第 177 行把目标 ENU 速度投影到给定方位；第 178 行乘以弦长到斜距的链式导数因子；第 179 行结束函数。[S]

**数学与单位。** 若传入的 `az` 确实是目标位置处沿地心角增大方向的局部方位，则

```text
D = 2R_e sin(sigma/2)
dD/dsigma = R_e cos(sigma/2)
dsigma/dt = v_along/R_e
dr/dD = D/r
```

从而

```text
dr/dt = (D/r) cos(sigma/2) v_along
```

第 178 行链式系数正确，量纲为无量纲×m/s=m/s。[M] 问题不在此链式系数，而在调用方传入的方位是否属于目标端正确切向。[M]

**边界。** `D=0` 时得到 `dr/dt=0`；`sigma=pi` 时 `cos(sigma/2)=0`，弦长在对跖点附近对地心角的一阶变化率为零，这与弦长函数在最大值处导数为零一致。[M] 若 `r=0` 会除零，但固定 `H>0` 的正常内部调用保证 `r>=2H`；独立辅助函数不可从文件外调用。[S][M]

**代码质量。** 函数单一职责清楚，但参数 `az` 的参考点和方向契约没有编码或注释，正是调用错误容易发生的原因。[S]

**复杂度。** `O(1)`。[S]

**建议。** 将参数改名为 `az_path_at_target_deg` 一类能表达参考点的名称，或让辅助函数直接接收站点/目标坐标并在内部计算正确梯度方向，减少错误传参空间。[S][M]

### 3.12 空行、分隔注释与函数边界（第 31、97—101、111—115、125—142、169—175 行）

**语句职责。** 这些行用于主函数与三个局部函数的视觉分隔、公式说明和推导说明，不改变执行结果。[S]

**代码质量。** 分隔清晰，局部函数顺序与调用关系基本一致；多普勒推导注释有助于审查链式法则。[S] 但第 139 行把 `dσ/dt` 直接写成以 `az` 投影的形式，却没有注明 `az` 必须是**目标端**切向方位，这掩盖了第 159—164 行的参考点错配。[S][M]

**建议。** 保留推导，但在每个几何方向量旁写明“定义在哪个点的局部 ENU 坐标系”；球面问题中同一条大圆在不同点的局部方位不可混用。[M]

## 6. 文件级结论与改进顺序

1. **先修正确性，再做重构。** 首要任务是用目标端路径梯度修正 `V08-P1-01`，并以解析结果对照数值中心差分；不要只用仿真端与 UKF 端“结果一致”作为正确性证明。[M][P]
2. **补齐最小接口防线。** 对 action、参数数量、输出数量、标量实数有限性、纬度范围、`D>=0` 和单位契约做显式校验。[S]
3. **抽取单段几何。** 建议私有辅助函数一次返回 `sigma,D,r` 以及目标端路径梯度方位，供 `group_range`、`full`、`doppler` 共用，消除重复。[S]
4. **明确模型可配置边界。** 若仍采用固定虚高模型，应由共享配置传入 `H`，并确保仿真与滤波读取同一值；如暂不改接口，至少把固定假设写入正式契约。[P]
5. **建立几何单元测试。** 应覆盖重合点、对跖点附近、赤道、同经线、一般斜向长距离、Tx/Rx 交换对称性、路径率有限差分和单位/符号约定。本文未执行这些测试，不把建议测试写成已验证事实。
