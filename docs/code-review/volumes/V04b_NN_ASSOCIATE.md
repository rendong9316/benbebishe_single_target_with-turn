## 3. 逐语句块审查

### 3.1 地理距离预筛选（第 22—26 行）

**语句职责。** 根据航迹生命期设置自适应地理门：新航迹 120 km，成熟航迹 60 km。[S]

**数学正确性。** 自适应门合理：新航迹位置估计不准，需要更大的搜索范围。[M]

**代码质量。** 硬编码 `120000` 和 `60000`（米）在参数外。建议提取为 `params.geo_gate_mature_m` 和 `params.geo_gate_new_m`。[S]

### 3.2 马氏距离门（第 28—29 行）

**语句职责。** `gate_threshold = gate_sigma^2 * 2`。[S]

**数学正确性。** 2D 卡方阈值。`gate_sigma` 默认 2.5（阈值 12.5），但入口常覆盖为 6（阈值 72）。[M]

**问题记录。** `V04b-P2-01`：阈值公式 `gate_sigma^2 * 2` 没有注释说明对应卡方自由度和门内概率。读者无法从代码直接理解 `gate_sigma=6` 对应 99.9999% 的门外概率。[S]

### 3.3 硬 Vr 门（第 31—37 行）

**语句职责。** probation 期放宽 Vr 门到 200 m/s，之后使用参数值。[S]

**数学正确性。** Vr 门假设真目标帧间径向速度变化远小于杂波的随机速度。注释称"真目标帧间 Vr 变化 < 5 m/s"，但这是经验假设，未在所有场景验证。[M]

**问题记录。** `V04b-P2-02`：`gate_vr_ms = max(gate_vr_ms, 200)` 在 probation 期（`life <= 8`）实际上禁用了 Vr 门（因为杂波 Vr 范围也是 [-200, 200]）。这与注释"不过滤"一致，但降低了杂波抑制能力。[S]

### 3.4 NN 关联主循环（第 39—76 行）

**语句职责。** 遍历所有检测，三级筛选后返回马氏距离最小的点迹。[S]

**数学正确性。**

```matlab
% 方位角包裹
if innov(2) > 180, innov(2) -= 360; end
mahal = innov' * (P_zz_2d \ innov);
```

标准 2D 马氏距离计算。[M]

**代码质量。** 地理距离和 Vr 门在马氏距离之前执行，是有效的早停优化。[S]

### 3.5 波门内点迹收集（第 78—109 行）

**语句职责。** 第二次遍历 `det_list`，收集所有通过三级筛选的点迹。[S]

**性能问题。** `det_list` 被遍历两次：第一次找 NN，第二次收集波门内点迹。两次遍历计算相同的地理距离和 Vr 门。应将三级筛选提取为辅助函数，或合并为单次遍历。[S]

**问题记录。** `V04b-P2-03`：双重遍历导致 `sphere_utils_haversine_distance` 被重复调用（每个点迹最多 2 次）。在杂波密集场景（每帧数百检测），此开销显著。[S]

### 3.6 `isequal(dp, best_det)` 语义（第 85 行）

**语句职责。** 跳过已选的 NN 点迹。[S]

**数学正确性。** `isequal` 比较两个结构体的所有字段。如果 `det_list` 中两个点迹结构体字段完全相同（极小概率），此比较会错误跳过。[S]

**问题记录。** `V04b-P3-01`：使用 `isequal(dp, best_det)` 而非索引比较。如果 `det_list` 中存在多个相同结构体的点迹（如复制品），会错误跳过。建议使用索引 `d ~= best_det_idx` 更安全。[S]

## 5. 文件级结论

- **数学可信度**：三级筛选逻辑正确，地理门自适应合理。Vr 门假设"真目标帧间 Vr 变化 < 5 m/s"在典型场景成立，但在高速机动或杂波 Vr 分布更广的场景可能失效。
- **代码质量**：硬编码地理门和 Vr 门参数；双重遍历导致重复计算；`isequal` 比较结构体而非索引。
- **性能**：`sphere_utils_haversine_distance` 被调用最多 2×N_detections 次。可提取三级筛选为单次遍历函数。
- **测试充分性**：端到端冒烟通过，但 Vr 门在不同机动场景的有效性未获独立验证。
- **剩余未验证项**：不同 `gate_sigma` 值对关联率的影响；Vr 门在高速机动场景的假阴性率。
