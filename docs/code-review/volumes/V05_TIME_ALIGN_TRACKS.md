## 3. 逐语句块审查

### 3.1 时间偏移读取（第 59 行）

**语句职责。** `dt_offset = params.time_offset_radar2_sec`（默认 13s）。[S]

### 3.2 CV 回退矩阵（第 95—99 行）

**语句职责。** 构造 4×4 CV 状态转移矩阵，`dt = -13` 表示回退。[S]

**数学正确性。**

```text
F(-13) = [1, -13, 0, 0;
          0,   1, 0, 0;
          0,   0, 1, -13;
          0,   0, 0,   1]
```

`x_new = F * x_old`：`lon_new = lon - 13 * v_lon`，`lat_new = lat - 13 * v_lat`。[M]

### 3.3 协方差传播（第 114—116 行）

**语句职责。** `P_new = F * P * F' + Q_dt`，`Q_dt = Q_base * (13/30)`。[S]

**数学正确性。**

```text
Q_dt = Q_base * (dt_abs / params.dt_sec) = Q_base * (13/30) ≈ 0.433 * Q_base
```

**问题记录。** `V05-P1-01`：回退的 Q 增量为前向预测的 43%。从随机过程角度，向前和向后传播的条件化方差应该相同（齐次马尔可夫过程）。但这里用 `Q_base * (dt_abs/dt_sec)` 而非 `Q_base`，意味着回退 13s 只增加前向 30s 过程的 43% 噪声。这在物理上不合理：回退增加的不确定度应该等于同等时间跨度的前向传播。[M]

**问题记录。** `V05-P1-02`：回退是确定性状态转移，不应增加过程噪声。Q 项的存在暗示"回退引入了新的不确定性"，但这与 CV 模型假设矛盾——CV 模型下回退是确定性的（已知速度和时刻）。若考虑模型误差（实际不是严格 CV），应使用与时间跨度成正比的 Q，而非与 `dt_sec` 的比例。[M]

### 3.4 航迹类型过滤（第 80—86 行）

**语句职责。** 跳过 type=7（终止）和 type=6（临时）航迹。[S]

**代码质量。** 硬编码 type 值。若未来航迹类型枚举变化，此处不会自动更新。[S]

### 3.5 浅拷贝语义（第 69 行）

**语句职责。** `aligned_snap = snap` 浅拷贝 cell 数组。[S]

**数学正确性。** MATLAB 的 cell 数组赋值是浅拷贝——`snap.trackList` 的 cell 引用不变，但后续 `aligned_snap.trackList{t} = trk` 修改了 cell 内容。[S]

**问题记录。** `V05-P2-01`：`trk = snap.trackList{t}` 是结构体值拷贝（MATLAB 结构体按值传递），后续 `trk.ukf.x = F * trk.ukf.x` 修改的是拷贝而非原始。写回 `aligned_snap.trackList{t} = trk` 正确更新了快照。但如果 `trk.ukf` 内含嵌套结构体（如 `ukf.cache`），浅拷贝可能导致嵌套字段共享引用。[S]

## 5. 文件级结论

- **数学可信度**：CV 回退公式正确。但 Q 缩放比例（13/30）和回退加 Q 的合理性存疑。
- **代码质量**：硬编码 type 值；浅拷贝嵌套结构体可能有引用共享风险。
- **性能**：每帧遍历所有航迹，`regularize_cov` 对每个航迹做一次 eig 分解。在 MC 循环中开销可忽略（航迹数少）。
- **测试充分性**：端到端冒烟通过，但回退协方差传播的正确性未获独立验证。
- **剩余未验证项**：回退 13s 对融合 RMSE 的影响；Q 缩放比例变化对融合一致性的影响。
