# 双基地外辐射源雷达逐帧仿真方案

## 1. 项目概述

对双基地（bistatic）外辐射源天波超视距雷达（OTH-SWR）进行逐帧 Monte Carlo 仿真：两部发射-接收对（Tx1/R1 与 Tx2/R2）分别接收目标回波，各自独立生成点迹、标校系统偏差、进行 UKF 滤波与航迹管理，最终对比航迹输出。

- **目标**：单架民航机，东海自南向北巡航（~230 m/s，约 828 km/h）
- **数据率**：dt = 30 s，总仿真时长约 1564 s（~52 帧）
- **检测模型**：Pd = 60%，Pfa = 10⁻³，分辨率单元数 1500（100 range × 15 azimuth）
- **杂波模型**：Poisson 杂波，λ = 1500 × 10⁻³ = 1.5 虚警/帧
- **跟踪器**：M/N = 3/5 滑窗起始，最近邻（NN）关联，K_loss = 4 终止，UKF 滤波

---

## 2. 双基地量测模型

### 2.1 核心物理量

双基地雷达与单基地的本质区别在于：发射站（Tx）和接收站（Rx）分置两地。

| 物理量 | 定义 | 公式 |
|--------|------|------|
| 群距离 Rg | Tx→目标 + 目标→Rx 距离之和 | Rg = r₀ + r₁ |
| 方位角 az | 接收站→目标的球面方位角 | az = atan2(Δlon·cos(lat), ...) |
| 双基地多普勒 vd | Tx方向 + Rx方向径向速度之和 | vd = v·u_Tx + v·u_Rx |

其中：
- **r₀** = Haversine(Tx, target)，发射站到目标的大圆距离
- **r₁** = Haversine(Rx, target)，接收站到目标的大圆距离

### 2.2 双基地反解（极坐标 → 经纬度）

已知群距离 Rg 和方位角 az，求解目标到接收站距离 r₁：

```
r₁ = (Rg² - d²) / [2 · (Rg - d · cos φ)]
```

其中：
- **d** = Haversine(Tx, Rx)，基线长度
- **φ** = az - azimuth(Rx→Tx)，方位差角

得到 r₁ 后，通过球面正算（Vincenty 直接法）由 Rx 坐标、方位角 az、距离 r₁ 计算目标经纬度。

### 2.3 威力量盖范围约束

雷达的实际地理探测范围受限于接收波束和 r₁ 距离：

| 约束 | 参数 | 说明 |
|------|------|------|
| 波束宽度 | 15° | 接收波束 3dB 宽度 |
| 距离近界 | r₁ ≥ 1000 km | 天波 OTH 跳距 |
| 距离远界 | r₁ ≤ 2000 km | 天波 OTH 最大探测距离 |
| 波束中心 | R1: 91.5°, R2: 83.5° | 独立指向目标区域（东海 ~129°E, 32°N） |

**关键设计决策**：两部雷达的波束中心必须分别计算，确保扇区在目标区域重叠。统一使用 90° 会导致 R2 扇区偏离目标区域。

### 2.4 群距离与地理距离的关系

- 群距离 Rg **不是** 2 × max(r₀, r₁)，而是 r₀ + r₁
- Tx→目标 和 目标→Rx 路径相互独立，群距离受两条路径共同影响
- 在 1000-2000 km 威力范围内，Rg 大约在 2000-4000 km 范围

## 3. 站点几何布局

```
发射站 Tx1: (108.0°E, 33.5°N) ── 接收站 R1: (112.0°E, 33.5°N)   [南阳]
发射站 Tx2: (110.0°E, 31.5°N) ── 接收站 R2: (114.0°E, 31.5°N)   [随州/孝感]

目标航迹: (127.5°E, 31.0°N) → (130.5°E, 33.0°N)
```

Tx-Rx 基线约 4° 经度（~380 km），Tx 位于 Rx 正西侧。

## 4. 全流程仿真步骤

### Phase 0：场景初始化

```
simulation_params() → 全局参数结构体
aircraft_trajectory_create() → 大圆航迹
aircraft_trajectory_generate() → 逐秒真值 [lon, lat, lon_rate, lat_rate, time]
radar_coverage_check() → 验证航迹在双方威力范围内
```

### Phase 1：系统偏差离线标定（ADS-B 模拟）

利用已知真值（模拟 ADS-B 数据），在航迹上选取 ~30 个标校点，估计每部雷达的距离偏置和方位偏置：

```
对每个标校点：
  1. 计算真实双基地量测（Rg_true, az_true）
  2. 叠加系统偏差 + 随机噪声 → (Rg_meas, az_meas)
  3. 记录偏差 ΔRg = Rg_meas - Rg_true, Δaz = az_meas - az_true
标定结果：
  dr_est = mean(ΔRg)   % 距离偏置估计
  da_est = mean(Δaz)   % 方位偏置估计
```

### Phase 2：逐帧主循环

每帧（k = 1, 2, ..., 52）每部雷达独立执行：

```
1. 目标位置插值（R1 和 R2 采样时刻差 13s）
2. 点迹生成（generate_frame_detections）：
   a. 威力量盖检查
   b. Pd 随机判决（60% 概率检出）
   c. 若检出：双基地真值 + 系统偏差 + 高斯噪声 → 目标点迹
   d. Poisson 杂波（λ=1.5）：群距离/方位/多普勒均匀随机
   e. 所有点迹标记 is_clutter 字段（仅用于统计，跟踪器不可见）
3. 偏差校正（减去 Phase 1 估计值）：
   Rg_corrected = Rg_meas - dr_est
   az_corrected = az_meas - da_est
4. 双基地反解（极坐标→经纬度），得到每个点迹的 lat/lon
5. 航迹处理（process_one_frame）：
   a. M/N 滑窗起始（不分目标/杂波，盲起）
   b. 最近邻关联（2D 波门：距离+方位，使用 P_zz 新息协方差）
   c. UKF 预测→更新（或纯预测）
   d. K_loss 终止判断 → LOST → 重新起始
```

### Phase 3：统计

```
compute_stats() → 总点迹数 / 目标检出数 / 虚警数
count_valid_frames() → 滤波有效帧数
```

### Phase 4：可视化

| 图号 | 内容 | 类型 |
|------|------|------|
| fig1 | 场景总览：站点、波束扇区、距离环、真实航迹 | geoplot 交互 |
| fig2a/b | R1/R2 三维点迹图（Range-Azimuth-Frame） | plot3 可旋转 |
| fig3 | 综合航迹图：真实航迹 + 两部雷达关联点迹 + UKF滤波 | geoplot + 复选框图层控制 |
| fig4 | 误差时间线（位置 RMSE + 检测/关联事件标记） | 双子图 |

### Phase 5：数据保存

输出 `results/simulation_YYYYMMDD_HHMMSS.mat`，包含：
- `detList_R1`, `detList_R2`：点迹列表（cell array）
- `trackState_R1`, `trackState_R2`：每帧跟踪器完整状态快照
- `calib`：标定结果（估计值 + 真值）
- `scene`：站点几何
- `truth`：真实航迹
- `params`：完整参数
- `s1`, `s2`：统计汇总

## 5. 跟踪器设计

### 5.1 状态机

```
UNINITIATED ──(有检测)──→ INITIATING ──(M/N满足)──→ TRACKING
     ↑                                                      │
     └──────────────(K_loss 连丢)── LOST ←──────────────────┘
```

### 5.2 M/N 航迹起始

- 滑窗长度 N = 5 帧，阈值 M = 3 帧
- **不对点迹做任何目标/杂波区分**——雷达不知道哪个是虚警
- 窗口内累计 ≥3 帧有检测（无论几个点迹）即触发起始
- 起始时保存首检测（first_det），用于两点差分初始化
- 两点差分初始化内置**速度合理性检验**：若估计速度 < 30 m/s 或 > 600 m/s，退化为零速单点初始化（使用当前帧点迹位置）

### 5.3 最近邻关联（NN）

对每个 TRACKING 帧：

```
1. UKF 预测步 → x_pred, P_pred, X_pred（Sigma 点集）
2. 各 Sigma 点通过量测模型 → Z_pred
3. 计算新息协方差 P_zz = R + Σ Wcᵢ·(Z_predᵢ - z_pred)(Z_predᵢ - z_pred)ᵀ
4. 仅用距离+方位二维（P_zz 的 2×2 子矩阵），排除多普勒
5. 对每个点迹：
   马氏距离 = innov_2dᵀ · inv(P_zz_2d) · innov_2d
   若 < gate_sigma² × 2（3σ 椭圆门，2 自由度 = 18），候选
6. 选择马氏距离最小的点迹关联
7. 若无点迹落入波门：纯预测（missed++）
```

**关键设计决策**：
- **排除多普勒**于波门之外。多普勒量测噪声仅 0.5 m/s，但 UKF 状态的速度估计误差对应多普勒误差可达 10-100 m/s，会导致马氏距离贡献 400-40000，远超阈值
- **使用 P_zz 而非 R**。P_zz 包含了状态不确定性在量测空间的投影，UKF 不确定时波门自动展宽，收敛后自动收紧
- 多普勒仍参与 UKF 更新步（提供速度信息），仅排除在波门外

### 5.4 UKF 滤波

| 组件 | 规格 |
|------|------|
| 状态向量 | x = [lon, lon_dot, lat, lat_dot]ᵀ (4×1) |
| 量测向量 | z = [Rg, az, vd]ᵀ (3×1) |
| 运动模型 | CV（匀速），F·x 线性转移 |
| 量测模型 | 双基地非线性（Haversine + 方位角 + 双基地多普勒） |
| Sigma 点 | 2n+1 = 9 点，α=1e-3, β=2, κ=0 |
| Q 过程噪声 | diag([1e-9, 1e-13, 1e-9, 1e-13]) × Q_scale (5e5) |
| R 量测噪声 | diag(10000², 1², 0.5²) |
| P 初始 | diag(0.5², 0.01², 0.5², 0.01²) |

## 7. 关键技术决策与经验教训

### 7.1 波束中心必须独立计算

两部雷达位置不同，指向同一目标区域的方位角不同：
- R1 (112°E, 33.5°N) → 目标区 (~129°E, 32°N): az ≈ 91.5°
- R2 (114°E, 31.5°N) → 目标区 (~129°E, 32°N): az ≈ 83.5°

统一使用 90° 会导致 R2 扇区偏离目标区域，覆盖率从 53/53 骤降。

### 7.2 跟踪器不可“面向结果编程”

**雷达不知道哪个点迹是杂波**。所有处理（M/N 起始、NN 关联、显示关联点迹）**必须对所有点迹一视同仁**。`is_clutter` 字段仅存在于检测结构体中供事后统计，跟踪器代码不得读取该字段做决策。

### 7.3 波门使用 P_zz 而非 R

| 方法 | 公式 | 问题 |
|------|------|------|
| ❌ R-only | `innovᵀ·R⁻¹·innov` | 状态不确定时波门过紧 |
| ✅ P_zz | `innovᵀ·P_zz⁻¹·innov` | 自动适应状态不确定性 |

### 7.4 多普勒不应参与波门判决

多普勒量测精度高（σ=0.5 m/s），但 UKF 速度估计误差投影到多普勒空间可达 10-100 m/s。将多普勒纳入波门会导致目标自己的点迹无法通过波门。

### 7.5 两点差分初始化 + 速度校验

单点初始化（零速假设）容易在杂波帧产生完全错误的位置估计。两点差分可利用时间间隔估算速度，并通过 30-600 m/s 范围校验排除明显的杂波配对。

### 7.6 随机种子管理

每部雷达使用独立随机流：
- 全局：`rng(42)` — 航迹生成、标定
- R1：`rng(42 + k)` — 点迹生成（每帧递增）
- R2：`rng(10042 + k)` — 点迹生成（偏移 10000 确保独立）

使用 `rng()` 全局状态而非 `RandStream` 对象，因为 `poissrnd` 不支持流对象。

## 8. 典型仿真结果

| 指标 | R1 (南阳) | R2 (随州) |
|------|-----------|-----------|
| 总帧数 | 52 | 52 |
| 目标检出 | 34 | 32 |
| 虚警点迹 | 81 | 94 |
| 滤波有效帧 | 46 (88%) | 44 (85%) |
| 目标正确关联 | 28/34 (82%) | 18/32 (56%) |
| 杂波误关联 | 2 | 3 |
| UKF 中位误差 | 9.8 km | — |
| 点迹中位误差 | 21.3 km | 15.6 km |
| UKF 提升倍数 | **2.2×** | — |

R1 结果充分验证了 UKF 的有效性（中位误差降至点迹的 46%）。R2 关联率偏低，属于该杂波密度（1.5/帧）下的随机现象——部分仿真中 M/N 起始窗首检测为杂波，导致初始航迹偏离目标。实际系统可通过多假设跟踪（MHT）或提高 Pd 来改善。
