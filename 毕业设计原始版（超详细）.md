# 双基地OTH-SWR单目标跟踪与航迹融合仿真系统 — 毕业设计原始版（超详细）


## 项目概述

本项目是一个完整的 **MATLAB 双基地高频天波超视距雷达（OTH-SWR）单目标跟踪与航迹融合仿真系统**。系统模拟两部异步采样的双基地雷达对空中飞机目标进行探测、跟踪与航迹融合的全流程，涵盖从航迹生成、量测仿真、系统偏差标定、UKF滤波跟踪、机动自适应、航迹融合到误差评估的完整流水线。

### 技术栈

- **语言**：MATLAB（纯 .m 脚本，无 Simulink 依赖，纯函数式编程 + action dispatcher 模式）
- **滤波**：UKF（无迹卡尔曼滤波，4 维状态 x 3 维量测，Cholesky Sigma 点生成）
- **自适应**：模糊推理（Sugeno-type 5 级隶属度）+ 新息序列机动检测 + Q 渐进提升
- **关联**：最近邻（NN）+ 概率数据关联（PDA，beta 权重计算）
- **融合**：SCC / Bar-Shalom-Campo / Covariance Intersection / Fast CI 四种算法
- **几何**：Haversine 球面距离 + 球面正算/反算 + 双基地几何反解 + 天波电离层传播模型
- **起始**：M/N 滑窗起始 + 多假设共识评分
- **评估**：RMSE / 中位误差 / 95%分位数 / NIS 一致性检验
- **调优**：397 组参数 x 10 随机种子的自动化全局搜索（tune_ukf_params.m）

## 一、物理背景与雷达体制


### 1.1 天波超视距雷达（OTH-SWR）

天波超视距雷达利用电离层对高频（3-30 MHz）电磁波的反射，实现数千公里的超视距探测。核心物理过程：
1. **发射**：地面发射站向电离层发射 HF 信号
2. **折射**：信号经电离层 F 层（高度约 250-400 km）折射后照射目标
3. **散射**：目标散射回波
4. **返回**：回波经电离层再次折射后到达接收站
这种一跳传播模式可实现 1000-3000 km 的超视距探测范围。

### 1.2 双基地体制

本仿真采用**双基地（Bistatic）雷达体制**：发射站（Tx）和接收站（Rx）地理分离。核心量测量包括：群距离 Rg = r0 + r1（Tx->目标 + 目标->Rx 的总路径长度）、方位角 az（Rx 观测目标的真北方位角）、双基地多普勒 vd = v*u_Tx + v*u_Rx（Tx方向 + Rx方向的径向速度投影和）。

**双基地反解公式**：已知群距离 Rg 和方位角 az，求解目标到接收站距离 r1：
```
r1 = (Rg^2 - d^2) / [2 * (Rg - d * cos(phi))]
```
其中 d = distance(Tx, Rx)（基线长度），phi = az - azimuth(Rx->Tx)（方位差角）。得到 r1 后，通过球面正算由 Rx 坐标、方位角 az、距离 r1 计算目标经纬度。

### 1.3 天波传播模型

与仿真端量测生成严格一致的天波几何模型：
1. **地心角** sigma = Haversine(站点, 目标)
2. **地表弦长** D = 2*R_e*sin(sigma/2)（替代大圆弧长）
3. **电离层虚高** H = 300 km（F 层等效反射高度）
4. **天波斜距** r = sqrt(D^2 + (2H)^2)（电离层双跳模型）
5. **群距离** Rg = r_tx + r_rx

### 1.4 站点几何布局

```
R1 接收站: (113.0E, 33.5N)  <-->  R1 发射站: (109.0E, 33.5N)
R2 接收站: (115.0E, 33.0N)  <-->  R2 发射站: (111.0E, 33.0N)

直线目标: (127.5E, 31.0N) --> (130.5E, 33.0N)  [东海->对马海峡, 230 m/s]
拐弯目标: (126.0E, 32.5N) --> (128.5E, 33.5N) --> (128.6E, 31.7N)  [~120deg拐角, 140 m/s]
```

## 二、系统架构与设计模式


### 2.1 整体架构

采用 **分层模块化架构 + Action Dispatcher 模式**，每个核心模块通过统一的 action 字符串入口调度内部的局部子函数。

顶层入口为 run_simulation.m / run_simulation_turn.m 两个主程序。核心模块层：config（参数配置）、simulation（场景仿真）、ukf（滤波核心）、tracker（航迹管理）、association（点迹关联）、initiation（航迹起始）、fusion（航迹融合）、registration（空间配准）。支撑模块层：evaluation（误差评估）、visualization（可视化）、io（数据I/O）、utils（球面几何）。所有模块通过全局函数调用耦合，参数通过 params 结构体传递，场景数据通过 cell 数组传递。

### 2.2 Action Dispatcher 模式

核心 UKF 模块采用 action dispatcher 设计模式：ukf_jichu.m 提供 6 个公共 action（create/init/prepare/update/predict/measurement），ukf_zishiying.m 提供 3 个公共 action（create/init/update），内部委托基础 UKF 完成 Kalman 数学后施加机动自适应后处理。

同样采用 dispatcher 模式的模块：
- evaluate_all.m: tracking_errors / fusion 两个 action
- plot_results.m: 7 种绘图模式（single_track/single_fusion/combined_tracks/...）
- plot_turn_spatial.m: 4 种模式（point_clouds/radar_compare/fusion_map/comprehensive）
- plot_turn_stats.m: 4 种模式（comparison/fusion_compare/rmse_bars/single_compare）
- measurement_simulator.m: create / measure 两个 action
- tracker_utils.m: init / process / finalize / stitch 四个 action

## 三、仿真主程序详解


### 3.1 直线航迹仿真 (run_simulation.m, 1237行) -- 9-Phase流水线


**Phase 0: 场景初始化**
- 加载参数（13个模块默认配置）；aircraft_trajectory_create基于航点列表+速度+步长生成航段模型
- aircraft_trajectory_interpolate(generate): 逐点批量采样，输出Nx5矩阵
- radar_coverage_check: 验证距离(1000-2000km)+波束扇区(beam_center +/- 7.5deg)
- 建立异步时间网格: R1=0s/30s/60s..., R2=13s/43s/73s...；构造真值结构体truthTraj

**Phase 1: ADS-B系统偏差标定**
- 加载278,139行ADS-B CSV数据；均匀采样5000个点
- 对每个威力范围内的ADS-B点: 真值极坐标反算->模拟量测(真值+偏差+噪声)->记录差值
- 样本均值估计: dr_est=mean(dr_list), da_est=mean(da_list)

**Phase 2: 原始点迹生成**
- 逐帧(n_frames约120)生成两部雷达的原始点迹
- generate_frame_detections内部: 覆盖率检查->Pd=0.6检测判决->高斯噪声叠加->泊松虚警生成
- 随机种子独立: R1=seed+k, R2=seed+10000+k (偏移10000确保独立)

**Phase 3: 时间对齐策略声明** (纯声明,不执行计算)
- 策略: 不在点迹级对齐(避免插值误差叠加), 延后到Phase 6航迹级对齐

**Phase 4: 偏差校正+双基地几何反解**
- 每个点迹: drange=prange-dr_est, daz=paz-da_est
- bistatic_inverse_solver(Rg,az,Tx,Rx)->(lat,lon)
- 同时保留原始偏差反解经纬度raw_lat/raw_lon用于对比评估

**Phase 5: 单目标航迹跟踪(UKF+PDA+模糊自适应Q)**
- 创建UKF模板: R1精密参数(Q_scale=1e5,gate_sigma=4.0), R2标准参数(Q_scale=2e5,gate_sigma=5.0)
- 调用single_track_runner(内部: M/N起始->prepare->NN关联->PDA加权->UKF更新->模糊自适应Q)
- 输出trackSnapshots(每帧航迹状态快照); 关联诊断: 关联率/NIS均值/门内比例

**Phase 6: 航迹级时间对齐**
- time_align_tracks: CV模型负时间外推, offset=13秒, 逆向回退约3km

**Phase 7: 航迹融合(四种算法)**
- 单目标直接1对1融合(R1航迹ID=1与R2航迹ID=1)
- 四种算法依次执行: SCC/BC/CI/FCI; 输出all_fused_snapshots{1..4}

**Phase 8: 定量误差评估**
- evaluate_all('fusion'): 4种融合+R1_only+R2_only=6行对比
- 指标: RMSE/中位误差/均值/95%分位数/UKF vs原始点迹改善百分比

**Phase 9: 可视化+数据保存**
- 5张图: 场景总览/R1点云/R2点云/单目标跟踪/融合对比
- 保存.mat文件

## 四、核心算法详解


### 4.1 UKF（无迹卡尔曼滤波）

**状态空间**: 4维 x = [lon, lon_dot, lat, lat_dot]^T（经度+经度变化率+纬度+纬度变化率）
**量测空间**: 3维 z = [Rg, az, vd]^T（双基地群距离、方位角、径向速度）
**UT参数**: alpha=1e-3(直线)或1e-1(拐弯), beta=2.0, kappa=0, lambda=alpha^2*(n+kappa)-n
**Sigma点**: Cholesky分解生成2n+1=9个点, 失败时退化处理为加扰动后重试

**每帧滤波循环（模块化流水线）**:
1. prepare: Sigma点->CV传播->x_pred/P_pred -> Z_pred->z_pred/P_zz（新息协方差）
2. nn_associate: 地理预筛(120/60km)+马氏距离(2D,仅距离+方位)->best_det+dets_in_gate
3. pda_weight: beta_i关联概率->3D加权新息向量innov_weighted
4. ukf_jichu update: P_xz->K->x_new=x_pred+K*innov->P_new->协方差正则化
5. apply_fuzzy_adapt: NIS滑动均值->模糊推理5级隶属度->Sugeno解模糊->Q缩放->EMA平滑

**关键设计决策**:
- 波门仅用距离+方位2D（排除多普勒）：多普勒量测噪声仅0.5 m/s，但UKF速度误差投影可达10-100 m/s
- 使用P_zz而非R：P_zz包含状态不确定性在量测空间的投影，UKF不确定时波门自动展宽
- 两点差分初始化+速度检验：利用时间间隔估算速度，10-2000 m/s范围内排除杂波配对
- 天波双跳模型：meas_to_latlon内部用经典双基地反解初值+30次迭代精化

### 4.2 机动自适应UKF (ukf_zishiying.m, 251行)

**三层自适应机制叠加**:

第一层 -- 模糊自适应Q（连续平滑调节）：
- NIS滑动均值(3帧窗口) -> NIS比值=avg/2.0
- 5级三角形隶属度(VS/S/M/L/VL) -> Sugeno加权平均
- 输出等级: Decrease(0.6)/SlightDecrease(0.8)/Maintain(1.0)/Increase(1.8)/RapidIncrease(3.0)
- EMA平滑(eta=0.10, V2调优后取值)

第二层 -- 机动检测（离散Q提升）：
- NIS趋势比较: 短时窗口(3帧) vs 长时窗口(全历史) NIS均值
- 判定条件: nis_short > nis_long * 1.25 && nis_short > 2.8 或 nis_long > 3.2
- Q渐进提升: 1.5 -> 2.3 -> 3.1 -> 3.5（非瞬间跳跃，避免协方差突变）
- 恢复: 连续4帧正常 -> 关闭机动标志
- 超时保护: 机动持续超过50帧 -> 强制退出

第三层 -- 机动预检测（suspect_counter）：
- 非机动状态下，用1.8x宽波门扫描备选量测
- 若有量测落入宽波门 -> suspect_counter++
- suspect_counter >= 2 -> 提前激活机动

### 4.3 航迹起始 -- M/N滑窗+共识评分

track_initiation.m 采用纯过程化 action dispatcher 模式：
- init action: 初始化长度为N=8的滑窗, 记录M=4, 速度检验范围30-600 m/s, 共识距离门限80km
- process action: 逐帧维护滑窗->当滑窗内>=M帧有点迹时触发起始->多假设配对(当前帧x历史帧)->Haversine距离->速度检验->共识评分(其他帧点迹是否靠近配对轨迹80km内)->选择最高评分对(评分>=1即成功)
- reset action: 丢失后清空窗口重新开始

### 4.4 航迹融合 -- 四种算法对比

| 算法 | 互协方差需求 | 权重计算 | 核心公式 | 特点 |
|------|-------------|----------|----------|------|
| SCC | 不需要 | w=0.5等效 | P_inv=P1_inv+P2_inv | 最简单,假设完全独立 |
| BC | 需维护P12 | 动态计算 | K=(P1-P12)/S, S=P1+P2-P12-P12^T | 最精确(P12准确时) |
| CI | 不需要 | fminbnd优化 | P_inv=w*P1_inv+(1-w)*P2_inv | 保守融合,不低估 |
| FCI | 不需要 | tr(P)倒数闭式解 | w=tr(P1)^-1/(tr(P1)^-1+tr(P2)^-1) | CI快速近似 |

BC方法的P12递推维护：预测步P12_pred=F*P12*F^T+Q_half（两传感器共享过程噪声,各计入1/2），更新步用迹收缩比sqrt(trace(P)/trace(P_pred))近似(I-KH)的压缩效果。稳定性约束：P12对角元不超过0.8*min(diag(P1))。

### 4.5 PDA -- 概率数据关联

当波门内有多个候选量测时，不硬选最佳，而用beta权重加权所有量测：
```
e_i = exp(-0.5 * mahal_2d(i))
beta_i = e_i / (b + sum(e))
b = 2*pi*lambda*sqrt(|S|)*(1-Pd*Pg)/(Pd*Pg)   # 杂波先验项
innov_weighted = sum(beta_i * innov_3d(i))
```

### 4.6 空间配准 -- 系统偏差估计

estimate_biases.m (528行) 实现双阶段算法：
- Stage 1 -- 直接最小二乘(LS): 逐帧计算(量测值-真值), 多帧均值消除零均值噪声->无偏估计
- Stage 2 -- EML精化: 在ECEF三维直角坐标系中建立代价函数, fmincon(SQP算法)数值优化
- 最终结论: LS结果优于fmincon精化, x_opt = x0 (采用LS估计)

## 五、UKF参数自动调优


### 5.1 调优策略

**两轮全局搜索**（tune_ukf_params.m）：

**V2四轮搜索**：
1. Round 1 (150组): Q_scale x gate_sigma 全局扫描, Q对数步进15个值, gate等步长10个值
2. Round 2 (80组): 最优附近加密 + R2/R1 Q比例(1.5/2.0/2.5)
3. Round 3 (49组): alpha(1e-4~1e-1) x P_pos_std(0.02~0.50)
4. Round 4 (40组): fuzzy_window(3~16) x ema_eta(0.05~0.50)

**评价指标**: Score = 0.5*RMSE_avg + 0.3*RMSE_turn + 0.2*Smoothness
**轻量化设计**: 预生成缓存(每种子的Phase 0-4只生成一次, 所有参数组合复用), 不画图/不保存/不运行融合

### 5.2 调优结果

**核心发现 -- 性能高原**: gate_sigma >= 3.0 且 Q_scale >= 3e4 后, 参数空间存在宽广高原(Score波动 < 3%)

**参数敏感性排序**:
1. gate_sigma (极高) -- <3.0时性能崩溃(-79%)
2. ukf_Q_scale (高) -- <3e4时滞后严重
3. ukf_alpha (低) -- 几乎无差异
4. P_pos_std (低) -- 0.10-0.20均可
5. fuzzy_window/eta (极低) -- 影响 <1%

**最终改善（拐弯场景）**:

| 指标 | 原始参数 | V2最优 | 改善 |
|------|----------|--------|------|
| R1 RMSE（自适应） | 18.2 km | 6.6 km | +63.7% |
| R2 RMSE（自适应） | 50.0 km | 8.2 km | +83.6% |
| 平均 RMSE | 34.1 km | 7.4 km | +78.3% |
| 平滑度 | 11.56 km | 3.55 km | +69.3% |

## 六、关联波门设计

**两步筛选策略**：第一步地理距离预筛(初始120km, UKF收敛后60km)，第二步马氏距离精筛(gate_sigma^2*2, 仅用距离+方位2D)。使用P_zz而非R作为波门尺度，使波门自动适配UKF的收敛状态。

**为什么排除径向速度于波门之外**：多普勒量测噪声仅0.5 m/s，但UKF速度估计误差投影到多普勒空间可达10-100 m/s，马氏距离贡献400-40000，使目标自己的点迹无法通过波门。

**拐弯场景的gate_sigma选择**：V2调优揭示拐弯场景需从默认2.0-2.5大幅放宽到7.0。原因是拐弯时CV模型预测偏差增大，严格门限导致有效量测被误拒->滤波器连续纯预测->误差发散。

## 七、技术决策与经验教训

1. **波束中心必须独立计算**：两部雷达位置不同，指向同一目标区域的方位角不同，统一使用90度会导致某站扇区偏离
2. **跟踪器不可面向结果编程**：雷达不知道哪个点迹是杂波，所有处理必须对所有点迹一视同仁，is_clutter字段仅供事后统计
3. **杂波生成空间选择**：初期在(Rg,az)空间均匀生成->反解后分布不均匀。修复：改为在(r1,az)空间均匀生成
4. **Haversine数值稳定性**：Sigma点极端位置时a值可能超出[0,1]->NaN传播->Cholesky分解失败。修复：a=clamp(a,0,1)
5. **速度检验阈值**：初始[30,600]m/s过严，放宽至[10,2000]m/s避免误拒合法目标对
6. **随机种子管理**：R1和R2使用偏移10000的独立种子空间，确保噪声独立

```
  |     +-> 偏差校正 + bistatic_inverse_solver -> detList_R1/R2
  |
  +-> single_track_runner/adaptive -> trackSnapshots_R1/R2
  |     +-> M/N起始->NN关联->PDA加权->UKF更新->模糊自适应Q
  |
  +-> time_align_tracks(R2) -> aligned_R2 (CV模型负时间外推)
  |
  +-> run_track_fusion() x4 -> all_fused_snapshots
  |     +-> SCC: P_inv=P1_inv+P2_inv
  |     +-> BC: K=(P1-P12)/(P1+P2-P12-P12^T)
  |     +-> CI: fminbnd优化w
  |     +-> FCI: w=tr(P1)^-1/(tr(P1)^-1+tr(P2)^-1)
  |

  +-> plot_* -> PNG / save() -> results/*.mat
```
