## 3. 逐语句块审查

### 3.1 时间设定（第 54—86 行）

**数学正确性。** 120 帧 × 30s = 3600s。R2 偏移 13s 产生交错采样（R1: 0,30,60... R2: 13,43,73...）。[M]

### 3.2 站点几何（第 98—114 行）

**语句职责。** 定义 Tx/Rx 经纬度。[S]

**数学正确性。** R1 Tx-Rx 基线约 4°（370 km），R1-R2 相距约 2°（185 km）。典型双基地配置。[M]

### 3.3 覆盖参数（第 130—151 行）

**语句职责。** 波束指向 92°/91°、宽度 15°、距离 1000-2000 km。[S]

**代码质量。** `range_min_km` 和 `range_max_km` 与 `range_min_m`/`range_max_m` 重复定义，后者由前者计算。若修改前者忘记修改后者会导致不一致。[S]

### 3.4 航迹参数（第 171—181 行）

**语句职责。** 两点航路、230 m/s 巡航、直线模式。[S]

**数学正确性。** 航路点 (127.5°E, 31.0°N) → (130.5°E, 33.0°N)，大圆距离约 380 km，飞行时间约 380000/230 ≈ 1652 s ≈ 55 帧。总时长 3600s 但目标飞行仅 1652s，后 1948s 目标已离开覆盖区。[M]

**问题记录。** `V01-P2-01`：航迹总飞行时间仅 55 帧，但仿真总帧数 120 帧。后 65 帧无目标，UKF 进入丢失终止逻辑。`tracker_K_loss=8` 意味着航迹在目标离开后约 8 帧（240s）终止，与实际飞行时间不冲突。但 MC 统计中这 65 帧的误差为空，需在评估中正确排除。[M]

### 3.5 量测噪声（第 201—216 行）

**语句职责。** R1 距离 7km/方位 0.35°，R2 距离 14km/方位 0.6°，径向速度 0.5m/s。[S]

**数学正确性。** R2 噪声约 R1 的 2 倍，异质传感器设计合理。[M]

### 3.6 系统偏差（第 235—241 行）

**语句职责。** R1 距离 +20km/方位 -3°，R2 距离 -15km/方位 +3.5°。[S]

**数学正确性。** 符号相反的偏差设计用于验证融合算法的偏差互校正能力。[M]

### 3.7 UKF 参数（第 266—299 行）

#### 3.7.1 UT 参数（第 271—277 行）

**语句职责。** `alpha=1e-2`, `beta=2.0`, `kappa=0.0`。[S]

**数学正确性。** `lam = alpha^2*(n+kappa)-n = 1e-4*4 - 4 = -3.9996`。`Wm(1) = lam/(n+lam) = -3.9996/0.0004 = -9999`。极端负权重使 UKF 退化为近似 EKF。[M]

**问题记录。** `V01-P1-01`：`alpha=1e-2` 导致负权重约 -9999，UKF 实质退化为近似 EKF。文档注释称"1e-2 优于 1e-3"，但 1e-3 时 `lam ≈ -3.9992`，`Wm(1) ≈ -9998`，差异极小。两个值都在数值不稳定区域。建议 `alpha=0.5` 或 `1.0`。[M]（已在 CR-ISSUE-016 登记）

#### 3.7.2 过程噪声 Q（第 287—291 行）

**语句职责。** R1 scale=1e5，R2 scale=2e5。[S]

**数学正确性。** `Q_R1 = diag([1e-4, 1e-8, 1e-4, 1e-8])`，`Q_R2 = diag([2e-4, 2e-8, 2e-4, 2e-8])`。速度方向噪声比位置小 1000 倍。[M]

#### 3.7.3 初始协方差 P（第 296—299 行）

**语句职责。** R1/R2 位置 0.05°，速度 R1 0.004°/s、R2 0.005°/s。[S]

**问题记录。** `V01-P2-02`：R1 和 R2 位置初始不确定度相同（0.05°），但 R2 噪声更大（2×R1）。R2 的初始 P 应更大以反映低精度先验。[M]

### 3.8 关联波门（第 304—312 行）

**语句职责。** R1/R2 `gate_sigma=6`，Vr 门 R1=20/R2=40。[S]

**数学正确性。** `gate_sigma=6` → 阈值 `72` → `chi2cdf(72,2)≈1`。[M]

**问题记录。** `V01-P1-02`：`gate_sigma=6` 在入口脚本中被覆盖（见 `run_simulation.m`），但全局默认 `gate_sigma=2.5`（阈值 12.5）在 `nn_associate.m:29` 使用。若入口未覆盖，NN 使用 12.5 而 UKF 内部使用 72，门限不一致。[S]

### 3.9 检测/虚警（第 399—428 行）

#### 3.9.2 分辨单元数（第 426—428 行）

**数学正确性。** 100 距离单元 × 15 方位单元 = 1500。每帧期望虚警 `1500 × 0.001 = 1.5`。[M]

### 3.10 PDA 参数（第 450—458 行）

**语句职责。** `pda_pd_gate=0.8647`，杂波密度 `1.5/(2000e3*15)`。[S]

## 5. 文件级结论

- **数学可信度**：参数设计整体合理（异质噪声、相反偏差、交错采样），但 `alpha=1e-2` 导致 UKF 数值不稳定，`Pg=0.8647` 与实际门限 72 不匹配，`P_d=1.0` 与 MC 表头 0.6 冲突。
- **代码质量**：IMM 配置 6 组重复；`range_min_km` 和 `range_min_m` 重复定义；全局 `gate_sigma=2.5` 依赖入口覆盖。
- **性能**：参数配置脚本无性能问题（仅赋值）。
- **测试充分性**：参数扫描脚本 `scan_Pi.m` 和 `scan_Q_scale.m` 直接重写 `simulation_params.m`，异常中断可能不能恢复。
- **剩余未验证项**：`alpha` 不同取值对转弯 RMSE 的影响；`Pg` 与真实门限不匹配对 PDA 权重的影响。
