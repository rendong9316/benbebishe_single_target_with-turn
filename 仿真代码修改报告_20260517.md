# 双基地外辐射源雷达仿真代码修改报告

**修改日期**：2026 年 5 月 17 日

---

## 一、修改背景

原始仿真代码在 seed=42 下运行存在三个层面问题：

1. **R2 航迹鬼影**：R2 UKF 中位误差 ~105 km，航迹系统偏离真值 ~100 km 且无法收敛
2. **数值崩溃**：UKF 更新过程中出现 NaN/Inf，Cholesky 分解失败导致程序中断
3. **真实地名残留**：代码注释及图中含南阳、随州/孝感、武汉、郑州等真实地理信息

根因定位经过：

- 杂波在 (Rg, az) 空间均匀生成 → 反解后地理分布不可控 → 改为 (r1, az) 空间
- R2 `range_meas`/`azimuth_meas` 字段未随 bias 校正更新 → `ukf_filter_init` 读到旧值 → 地理位置与量测空间不一致
- Haversine 公式中 `a` 值在 Sigma 点极端位置时浮点超 [0,1] → `sqrt(1-a)` 复数 → NaN 传播
- 速度检验 [30,600] m/s 对位置噪声 ~30 km 而言过严 → 合法目标对 30% 被误拒
- M/N 起始器首检测为杂波（~10% 概率）→ 航迹初始化在错误位置且无法恢复

## 三、详细修改说明

### 3.1 杂波生成：由 (Rg, az) 改为 (r1, az) 空间

**文件**：`simulation/generate_frame_detections.m`

**问题**：杂波在双基地群距离 (Rg, az) 空间均匀生成，经反解后地理分布非均匀，部分落在威力范围外。

**修改**：

```matlab
% 旧：Rg 空间均匀 → 分布不可控
fake_range = Rg_min + rand() * (Rg_max - Rg_min);  % Rg 均匀
fake_az = beam_center - half_beam + rand() * beam_width;

% 新：(r1, az) 空间均匀 → 威力范围内均匀分布
fake_r1 = range_min_m + rand() * (range_max_m - range_min_m);  % r1 均匀
fake_az = beam_center - half_beam + rand() * beam_width;
[clut_lon, clut_lat] = sphere_utils_destination_point(rx_lon, rx_lat, fake_r1, fake_az);
r0 = sphere_utils_haversine_distance(tx_lon, tx_lat, clut_lon, clut_lat);
fake_Rg = r0 + fake_r1;

% 掺入真实系统偏差：使主循环偏差校正后 drange ≈ fake_Rg, daz ≈ fake_az
det.prange = fake_Rg + range_bias;
det.paz = fake_az + az_bias;
```

杂波经纬度由 (r1, az) 直接正算得到，存入 `det.lat`/`det.lon`。主循环中通过 `isfield(det,'lat') && ~isnan(det.lat)` 判断跳过反解。

### 3.2 航迹起始：共识配对 + 质量确认

**文件**：`run_simulation.m`, `process_one_frame` 函数

**原有策略**：M/N=3/5 滑窗计数，触发后用首帧首检测 + 当前帧首检测做两点差分初始化 UKF。

**问题**：首帧首检测有 ~10% 概率为杂波（Pd=60% 时首帧无目标检测），导致初始位置偏差 ~100 km。UKF 收敛前，P_zz 波门宽达 ~168 km，杂波持续被关联，航迹锁定在鬼影上。

**新策略**：

#### (a) 共识配对

M/N 触发后，不立即起始，而是遍历窗内所有帧-帧检测配对：

1. 当前帧首检测 `det_now` 与之前各帧每个检测 `dp` 配对
2. 对每个配对，统计窗内其他帧有多少检测同时落在 `dp` 和 `det_now` 的 **80 km** 半径内（共识支持度）
3. 选支持度最高的配对
4. **仅当支持度 ≥ 1（即至少 1 个其他帧的检测支持该配对）时才起始**
5. 无共识配对则暂不起始，继续滑窗等待

#### (b) 地理距离预筛选

起始后前 **15 帧**（确认期），在 Mahalanobis 波门之前增加固定地理距离预筛选：

```matlab
use_geo_gate = (t.life <= 15);
if use_geo_gate
    geo_err = haversine(x_pred(1), x_pred(3), dp.lon, dp.lat);
    if geo_err > 80000, continue; end  % 80 km 地理门
end
```

原理：UKF 初始阶段 P 较大，P_zz 波门宽达 ~168 km（受状态不确定性主导），杂波极易落入。固定 80 km 地理门比宽 Mahalanobis 门更严格，防止鬼影建立。

#### (c) 航迹质量确认

```matlab
% 关联成功：quality = min(quality + 1, 5)
% 纯预测：  quality = max(quality - 1, 0)
% 确认期15帧内，若 quality ≤ 0 且连续2帧无关联 → 判定起始失败 → LOST
if t.life <= 15 && t.quality <= 0 && t.missed >= 2
    t.status = 'LOST';
end
```

### 3.3 Bug 修复详情

#### Bug 1：R2 检测的 `range_meas`/`azimuth_meas` 未更新

**位置**：`run_simulation.m` 第 168-169 行（R2 偏差校正循环）

上次编辑时误删了以下两行：
```matlab
det_r2(d).range_meas = Rgc;
det_r2(d).azimuth_meas = azc;
```

**影响**：`ukf_filter_init` 通过 `meas.range_meas` 获取量测值进行 UKF 初始化，读到的是 `generate_frame_detections` 中预设的 bias 校正前的旧值，与 `det.lat`/`det.lon`（由 bias 校正后的 `Rgc`/`azc` 反解得到）不一致。导致 UKF 初始状态与检测位置偏差 ~0.66°（约 73 km）。

已恢复该两行赋值。

#### Bug 2：R2 时间戳双重偏移

**位置**：`run_simulation.m` 第 158 行（R2 `generate_frame_detections` 调用）

原代码：
```matlab
det_r2 = generate_frame_detections(..., k, t2_grid(k), params.time_offset_radar2_sec, ...);
```

其中 `t2_grid` 已是偏移 13s 的序列（13, 43, 73...），函数内部又执行 `actual_time = time_sec + t_offset = t2_grid(k) + 13`，导致 R2 时间戳为 26, 56, 86...。

已移除 `t_offset` 参数及函数内部 `actual_time` 计算。

#### Bug 3：Haversine 公式 NaN

**位置**：`ukf/ukf_measurement_model.m`, `ukf/ukf_meas_to_latlon.m`

UKF Sigma 点可达极端经纬度，浮点误差使 Haversine 中间量 `a = sin²(Δlat/2) + cos(lat1)cos(lat2)sin²(Δlon/2)` 略超 [0,1]（典型值 1+1e-16），导致 `sqrt(1-a)` 为复数，经 `atan2` 产生 NaN，继而污染 P_zz → Kalman 增益 → 状态向量 → Cholesky 崩溃。

修复：`a = max(0, min(1, a))`.

#### Bug 4：速度检验阈值不适应位置噪声

**位置**：`ukf/ukf_filter_init.m` 第 78 行

两点位置差分估计速度：

- 相邻帧（dt=30s）：σ_pos ≈ 30 km（距离+方位合成），σ_speed = √2 × 30 km / 30 s ≈ 1414 m/s
- 间隔 4 帧（dt=120s）：σ_speed ≈ √2 × 30 km / 120 s ≈ 354 m/s

原阈值 [30, 600] m/s 对相邻帧合法目标对（真速 ~230 m/s，3σ 范围内 [230−4242, 230+4242]）有 ~30% 误拒概率。

修改为 [10, 2000] m/s，覆盖所有可能帧间隔。在共识配对阶段已彻底移除了速度检验。

## 六、已知局限与后续建议

1. **单次运行随机性**：当前为 seed=42 的单次仿真。Pd=60% + Poisson 杂波下不同随机种子结果有差异。建议后续做 Monte Carlo（≥100 次）统计两站正确关联率、RMSE 分布、收敛概率。

2. **R2 确认期波动**：R2 在确认期内偶有发散—重起始循环。根本原因是位置噪声 ~30 km 时 UKF 收敛需要较多帧数（~10 帧以上），而确认期内可用目标检测仅 ~6 帧（Pd=60%）。后续可尝试自适应确认期长度。

3. **两站融合**：当前 R1 和 R2 独立滤波。导师要求融合两条航迹为一条更精确的轨迹，此为本毕设核心贡献点，尚未实现。

4. **多误差量级测试**：导师要求测试不同噪声水平下的算法性能边界。当前仅 σ_range=10km, σ_az=1° 一组参数。
