## 2. 文件职责与接口契约

### 2.1 职责

自适应 UKF 调度器，封装 `ukf_jichu` 并附加机动检测和模糊自适应 Q。它不实现任何滤波数学，只在 `update` 动作中委托 `ukf_jichu('update', ...)` 完成 Kalman 更新，再调用 `apply_maneuver_adapt_post` 根据 NIS 趋势和宽门预检测调整 `ukf.Q`。[S][P]

### 2.2 输入输出与单位契约

| action | 输入 | 输出 |
|---|---|---|
| `create` | params, 雷达/Tx 位置, dt | 带机动检测字段的 UKF 模板 |
| `init` | 模板, 量测 | 初始化状态 |
| `prepare` | 模板 | 委托 `ukf_jichu` |
| `update` | 模板, PDA 加权新息 | `lon, lat, ukf`（含更新后的 Q） |

关键新增字段：`maneuver_active`, `maneuver_counter`, `maneuver_recovery`, `suspect_counter`, `innov_history`, `Q_ema`, `Q_base`, `last_innov`, `last_det_list`, `life_count`, `nis_history`。[S]

## 3. 逐语句块审查

### 3.1 主调度器（第 17—48 行）

#### 3.1.1 create/init/prepare（第 18—36 行）

**语句职责。** 委托 `ukf_jichu` 创建基础模板，添加机动检测字段。[S]

**代码质量。** `filter_type='zishiying'` 供 `ukf_dispatch` 路由，合理。[S]

#### 3.1.2 update（第 37—44 行）

**语句职责。** 委托 `ukf_jichu('update', ...)` 完成 Kalman 更新，若新息非空则调用 `apply_maneuver_adapt_post`。[S]

**数学正确性。** 自适应 Q 在 Kalman 更新后施加（post-hoc），意味着更新使用的 `Q` 是上一帧的自适应结果。这避免了"用当前帧新息预测 Q 再更新"的循环依赖，是合理的因果设计。[M]

**问题记录。** `V03b-P2-01`：`update` 动作签名与 `ukf_jichu('update')` 不同——它额外接收 `innov, z_pred, Z_pred, X_pred, x_pred, P_pred, P_zz, params`，但实际只使用 `innov`（第 39 行 `varargin{2}`）。其余参数仅被 `apply_maneuver_adapt_post` 间接通过 `ukf.cache` 访问。签名过长且部分参数未直接使用，易误导调用者。[S]

### 3.2 `apply_maneuver_adapt_post` — 机动自适应后处理（第 56—254 行）

#### 3.2.1 成熟度检查（第 57—65 行）

**语句职责。** 航迹未成熟（`life_count < mature_frames`）时跳过自适应。[S]

**数学正确性。** 初期滤波尚未收敛，NIS 统计无意义，跳过自适应合理。[M]

#### 3.2.2 字段初始化（第 67—78 行）

**语句职责。** 确保机动检测字段存在，初始化 `Q_ema=1.0` 和 `Q_base`。[S]

**数学正确性。** `Q_ema` 初始为 1.0 表示无自适应缩放。[M]

#### 3.2.3 NIS 历史检查（第 80—83 行）

**语句职责。** 要求至少 3 个 NIS 历史样本。[S]

**代码质量。** 检查 `nis_history` 字段存在性和长度。[S]

#### 3.2.4 阈值参数读取（第 85—95 行）

**语句职责。** 从 `params` 读取机动检测阈值，有硬编码默认值。[S]

**数学正确性。** 默认值：`nis_ratio_thresh=1.25`（短时 NIS 是长时的 1.25 倍以上）、`nis_short_abs=2.8`（短时绝对阈值）、`nis_long_abs=3.2`（长时绝对阈值）、`recovery_frames=4`（连续 4 帧无机动才恢复）、`max_duration=50`（机动最长持续 50 帧）。[M]

**问题记录。** `V03b-P2-02`：`nis_short` 使用最近 3 帧均值，`nis_long` 使用全部历史均值。若历史包含大量非机动帧，`nis_long` 会被稀释，导致 `nis_ratio_thresh` 条件更容易满足。建议 `nis_long` 也使用滑动窗口。[M]

#### 3.2.5 NIS 趋势机动检测（第 97—131 行）

**语句职责。** 比较短时和长时 NIS 均值，触发/恢复机动标记。[S]

**数学正确性。**

```text
nis_short = mean(nis_history[end-2:end])
nis_long = mean(nis_history)
maneuver = (nis_short > nis_long * 1.25 && nis_short > 2.8) || nis_long > 3.2
```

双条件设计合理：既需要趋势突变（短时显著大于长时），也需要绝对超标。[M]

**代码质量。** 状态机逻辑：`maneuver_active` 为真时递增计数器，连续无机动帧达到 `recovery_frames` 则恢复；超过 `max_duration` 强制恢复。[S]

**问题记录。** `V03b-P2-03`：`nis_history` 是全局均值，包含机动和非机动帧。若机动持续 50 帧，`nis_long` 会被逐渐拉高，使得 `nis_short/nis_long` 比值下降，可能提前结束机动检测。[M]

#### 3.2.6 宽门预检测（第 133—176 行）

**语句职责。** 在非机动状态下，扫描宽门（`wide_gate_mult=1.8` 倍）内是否有量测落入，若有则累加 `suspect_counter`。[S]

**数学正确性。**

```text
wide_gate = (gate_sigma * 1.8)^2 * 2
```

若 `gate_sigma=6`，宽门为 `(10.8)^2 * 2 = 233.28`。[M]

**代码质量。**

```matlab
if inno(2) > 180, inno(2) = inno(2) - 360;
elseif inno(2) < -180, inno(2) = inno(2) + 360; end
```

方位角包裹正确。但 `inno(2)` 是方位差，直接用于 2D 马氏距离时，方位维单位是 deg，距离维单位是 m，两者量纲不同却直接放入同一马氏距离计算。[M]

**问题记录。** `V03b-P1-01`：宽门预检测使用 `[drange; daz]` 的 2D 新息计算马氏距离，但 `P_zz(1:2,1:2)` 中 `(1,1)` 是 m²、`(2,2)` 是 deg²，量纲不一致的马氏距离无统计意义。[M]
`V03b-P2-04`：`suspect_counter` 初始值可能为负（第 141-143 行），逻辑上不合理。[S]

#### 3.2.7 新息历史记录（第 178—184 行）

**语句职责。** 记录 `last_innov` 到 `innov_history`，保持最多 10 帧滑动窗口。[S]

**代码质量。** `innov_history` 是 cell 数组，动态增长。MATLAB 的 cell 数组末尾追加是 `O(1)` 摊销，但头部删除 `innov_history(1)=[]` 是 `O(n)`。[S]

**问题记录。** `V03b-P2-05`：`innov_history` 和 `nis_history` 是两个独立的滑动窗口，但 `nis_history` 是全局累积的（见 3.2.5），两者语义不一致。[S]

#### 3.2.8 模糊自适应 Q（第 186—209 行）

**语句职责。** 基于 NIS 均值比率，通过 5 个三角形隶属函数计算 Q 缩放因子。[S]

**数学正确性。**

```text
nis_ratio = nis_avg / 2.0
```

除以 2.0 是经验缩放，使 NIS 均值映射到隶属函数支撑区间。[M]

5 个三角形隶属函数：

| 模糊集 | 支撑区间 | 峰值 |
|---|---|---|
| VS（非常小） | [0, 0.4] | 0.0 |
| S（小） | [0.2, 0.8] | 0.5 |
| M（中） | [0.6, 1.5] | 1.0 |
| L（大） | [1.3, 3.0] | 2.0 |
| VL（非常大） | [2.5, 4.0] | 4.0 |

重心法去模糊：

```text
factor = Σ mu_i * out_i / Σ mu_i
out = [0.6, 0.8, 1.0, 1.8, 3.0]
```

标准模糊推理结构。[M]

**代码质量。** 重叠区间设计合理（相邻模糊集有 0.2 的重叠），避免跳跃。[S]

**问题记录。** `V03b-P1-02`：隶属函数支撑区间和输出值均为经验设计，无理论依据。当 `nis_ratio` 超出 [0, 4] 范围时，所有隶属函数为 0，`total_mu < 1e-10` 回退到 `factor=1.0`，即无自适应。这可能掩盖 `nis_ratio > 4` 时的严重机动。[M]

#### 3.2.9 机动 Q 提升（第 211—231 行）

**语句职责。** 机动持续期间渐进提升 Q：0-5 帧从 1.5 升到 2.3，5-15 帧升到 3.0，15 帧后到 3.5。[S]

**数学正确性。** 渐进提升避免 Q 突变导致的滤波震荡。[M]

**问题记录。** `V03b-P2-06`：`factor_raw = max(factor_fuzzy, maneuver_target)` 取两者较大值，但 fuzzy 自适应在非机动时也有效。当两者冲突时，机动提升主导。设计合理但缺乏灵敏度分析。[S]

#### 3.2.10 因子钳制与 EMA 平滑（第 233—244 行）

**语句职责。** 钳制 `factor_raw` 到 [0.5, 4.0]，EMA 平滑后更新 `Q`。[S]

**数学正确性。**

```text
Q_ema_new = 0.2 * factor_raw + 0.8 * Q_ema_old
Q = Q_base * Q_ema  (若 |Q_ema - 1.0| >= 0.05)
```

EMA 系数 0.2 使自适应有适度惯性，避免对单帧异常过度响应。[M]

**问题记录。** `V03b-P2-07`：`abs(Q_ema - 1.0) < 0.05` 时直接设 `Q = Q_base`，忽略了 Q_ema 可能已经在 0.95-1.05 范围内缓慢漂移的情况。这导致小幅持续自适应被静默忽略。[S]

#### 3.2.11 机动方向记录（第 246—253 行）

**语句职责。** 记录机动方向供诊断。[S]

**代码质量。** 仅用于诊断，不影响滤波。[S]

### 3.12 `trimf_val_maneuver` — 三角形隶属函数（第 261—269 行）

**语句职责。** 标准三角形隶属函数求值。[S]

**数学正确性。** `trimf(x, a, b, c)`：`x≤a` 或 `x≥c` 时为 0，`a<x<b` 时线性上升到 1，`b≤x<c` 时线性下降到 0。[M]

**代码质量。** 简洁正确。[S]

## 5. 文件级结论

- **数学可信度**：模糊自适应 Q 的三角隶属函数和重心去模糊结构正确，但支撑区间、输出值和 NIS 比率缩放均为经验设计。宽门预检测中量纲不一致的马氏距离无统计意义。NIS 全局均值与短时均值比较在机动持续期间可能退化。
- **代码质量**：状态机逻辑完整，但接口签名过长且部分参数未使用；`nis_history` 和 `innov_history` 语义不一致；`suspect_counter` 负值处理不合理。
- **性能**：`innov_history` 头部删除 `O(n)`，但窗口仅 10，影响可忽略。宽门预检测逐帧遍历 `last_det_list`，在杂波密集场景可能较慢。
- **测试充分性**：未获独立函数级运行验证。模糊推理的灵敏度、NIS 趋势检测的 ROC 曲线、EMA 系数选择均需要 Monte Carlo 扫描。
- **剩余未验证项**：不同 `ema_eta` 和 `nis_ratio_thresh` 对转弯 RMSE 的影响；宽门预检测的假阳性率。
